Работа с пользователем

Получаем данные пользователя и связываем их с базой данных

Получение текущего пользователя

После входа в систему нам нужно узнать, кто вошёл. Для этого используем функцию getUser():

JavaScript — получение пользователя 
// Получаем текущего пользователя
const { data: { user }, error } = await supabase.auth.getUser()

if (user) {
  console.log('Пользователь вошёл:', user.email)
} else {
  console.log('Пользователь не авторизован')
}
Простыми словами: getUser() — это как спросить у охранника "Кто сейчас внутри?"
Что внутри объекта user?

Объект user содержит много полезной информации:

JavaScript — свойства пользователя 
// Пример объекта user
{
  id: "a1b2c3d4-e5f6-7890-abcd-ef1234567890",  // Уникальный ID
  email: "vasya@mail.ru",                        // Email
  created_at: "2024-01-15T10:30:00Z",           // Дата регистрации

  // Если вошли через Google/GitHub:
  user_metadata: {
    avatar_url: "https://...",                   // Аватарка
    full_name: "Вася Пупкин",                    // Имя
    name: "Вася"                                 // Короткое имя
  }
}
Свойство Что это Пример
user.id Уникальный идентификатор a1b2c3d4-e5f6...
user.email Email пользователя vasya@mail.ru
user.created_at Когда зарегистрировался 2024-01-15
user.user_metadata.avatar_url Ссылка на аватарку https://...
user.user_metadata.full_name Полное имя Вася Пупкин
Важно: user_metadata заполняется только при входе через Google/GitHub. При регистрации по email там будет пусто.

async function showUserInfo() {

    // Получаем элементы

Добавляем user_id в таблицу

Чтобы каждый пользователь видел только свои данные, нужно связать записи с пользователем. Добавим колонку user_id:

SQL — добавляем колонку user_id 
-- Добавляем колонку user_id в таблицу transactions
ALTER TABLE transactions
ADD COLUMN user_id UUID REFERENCES auth.users(id);

-- Для новых таблиц сразу добавляйте user_id:
CREATE TABLE transactions (
  id SERIAL PRIMARY KEY,
  amount INTEGER NOT NULL,
  description TEXT,
  category TEXT,
  created_at TIMESTAMP DEFAULT NOW(),
  user_id UUID REFERENCES auth.users(id)  -- Связь с пользователем
);
UUID — это формат ID пользователя (длинная строка с буквами и цифрами).
REFERENCES auth.users(id) — связывает нашу таблицу с таблицей пользователей.
Обновляем Money Tracker

Теперь при добавлении транзакции сохраняем ID пользователя:

JavaScript — добавление транзакции с user_id 
async function addTransaction(amount, description, category) {
  // Получаем текущего пользователя
  const { data: { user } } = await supabase.auth.getUser()

  if (!user) {
    alert('Сначала войдите в систему!')
    return
  }

  // Добавляем транзакцию с user_id
  const { data, error } = await supabase
    .from('transactions')
    .insert({
      amount: amount,
      description: description,
      category: category,
      user_id: user.id  // Важно! Сохраняем ID пользователя
    })

  if (error) {
    console.error('Ошибка:', error)
  } else {
    console.log('Транзакция добавлена!')
  }
}
Результат: Теперь каждая транзакция "помнит", кто её создал!
RLS — защита данных пользователя

RLS (Row Level Security) — это правила, которые говорят: "каждый человек может видеть только свои данные".

Зачем это нужно? Без RLS любой пользователь может увидеть ВСЕ записи в таблице. С RLS — только свои.
SQL — включаем RLS и создаём политики 
-- 1. Включаем RLS для таблицы
ALTER TABLE transactions ENABLE ROW LEVEL SECURITY;

-- 2. Политика для чтения: видеть только свои записи
CREATE POLICY "Пользователи видят свои транзакции"
ON transactions FOR SELECT
USING (user_id = auth.uid());

-- 3. Политика для добавления: создавать только свои записи
CREATE POLICY "Пользователи создают свои транзакции"
ON transactions FOR INSERT
WITH CHECK (user_id = auth.uid());

-- 4. Политика для обновления: редактировать только свои
CREATE POLICY "Пользователи редактируют свои транзакции"
ON transactions FOR UPDATE
USING (user_id = auth.uid());

-- 5. Политика для удаления: удалять только свои
CREATE POLICY "Пользователи удаляют свои транзакции"
ON transactions FOR DELETE
USING (user_id = auth.uid());

Что означает каждая часть:

  • auth.uid() — ID текущего авторизованного пользователя
  • user_id = auth.uid() — показывать только те записи, где user_id совпадает с текущим пользователем
  • FOR SELECT — правило для чтения данных
  • FOR INSERT — правило для добавления данных
Теперь данные защищены! Вася не сможет увидеть транзакции Маши, даже если очень захочет.
Страница профиля пользователя

Создадим простую страницу профиля, где пользователь может увидеть свои данные:

HTML — profile.html 
<div class="profile-page">
  <h1>Мой профиль</h1>

  <div class="profile-card">
    <img id="profileAvatar" src="" alt="Аватар" class="profile-avatar">

    <div class="profile-info">
      <p><strong>Имя:</strong> <span id="profileName"></span></p>
      <p><strong>Email:</strong> <span id="profileEmail"></span></p>
      <p><strong>Дата регистрации:</strong> <span id="profileDate"></span></p>
    </div>

    <button id="logoutBtn" class="btn-logout">Выйти из аккаунта</button>
  </div>
</div>
JavaScript — загрузка профиля 
async function loadProfile() {
  const { data: { user } } = await supabase.auth.getUser()

  if (!user) {
    // Если не авторизован — перенаправляем на страницу входа
    window.location.href = 'login.html'
    return
  }

  // Заполняем данные профиля
  document.getElementById('profileAvatar').src =
    user.user_metadata?.avatar_url || 'default-avatar.png'

  document.getElementById('profileName').textContent =
    user.user_metadata?.full_name || 'Не указано'

  document.getElementById('profileEmail').textContent = user.email

  // Форматируем дату регистрации
  const date = new Date(user.created_at)
  document.getElementById('profileDate').textContent =
    date.toLocaleDateString('ru-RU')
}

// Выход из аккаунта
document.getElementById('logoutBtn').addEventListener('click', async () => {
  await supabase.auth.signOut()
  window.location.href = 'login.html'
})

// Загружаем профиль при открытии страницы
loadProfile()
