Требования к проекту

Обязательный функционал:

  • Страница регистрации (email + пароль)
  • Страница входа (email + пароль)
  • Social Login (Google или GitHub)
  • Защищённый dashboard (только для авторизованных)
  • Транзакции привязаны к пользователю (user_id)
  • Кнопка выхода (Logout)
  • RLS политики настроены
Пошаговое руководство
Подсказка: Используйте Cursor для генерации кода! Просто описывайте что нужно сделать.

Шаг 1: Изменения в базе данных

SQL — Добавить user_id в таблицу transactions 
-- Добавляем колонку user_id
ALTER TABLE transactions
ADD COLUMN user_id UUID REFERENCES auth.users(id);

-- Делаем её обязательной для новых записей
-- (старые записи можно удалить или обновить)

Шаг 2: Настройка RLS политик

SQL — RLS политики для transactions 
-- Включаем RLS
ALTER TABLE transactions ENABLE ROW LEVEL SECURITY;

-- Политика на чтение: видим только свои транзакции
CREATE POLICY "Users can view own transactions"
ON transactions FOR SELECT
USING (auth.uid() = user_id);

-- Политика на создание: можем создавать только для себя
CREATE POLICY "Users can create own transactions"
ON transactions FOR INSERT
WITH CHECK (auth.uid() = user_id);

-- Политика на обновление: можем менять только свои
CREATE POLICY "Users can update own transactions"
ON transactions FOR UPDATE
USING (auth.uid() = user_id);

-- Политика на удаление: можем удалять только свои
CREATE POLICY "Users can delete own transactions"
ON transactions FOR DELETE
USING (auth.uid() = user_id);

Шаг 3: Создание страниц авторизации

Попросите Cursor создать:

  • /login — страница входа
  • /register — страница регистрации
  • Middleware для защиты dashboard

Шаг 4: Интеграция с транзакциями

При создании транзакции добавляйте user_id текущего пользователя:

JavaScript — создание транзакции с user_id 
const { data: { user } } = await supabase.auth.getUser();

const { data, error } = await supabase
  .from('transactions')
  .insert({
    amount: 50000,
    category: 'Зарплата',
    type: 'income',
    user_id: user.id  // Привязываем к пользователю!
  });

Шаг 5: Добавление кнопки выхода

JavaScript — выход из системы 
const handleLogout = async () => {
  await supabase.auth.signOut();
  router.push('/login');
};
Изменения в базе данных

Обновлённая структура таблицы transactions:

Поле Тип Описание
id int8, Primary Key Уникальный ID
amount numeric Сумма
type text "income" или "expense"
category text Категория
user_id UUID, FK -> auth.users НОВОЕ! ID владельца
created_at timestamptz Дата создания
Важно! Если у вас есть старые транзакции без user_id — их нужно удалить или обновить. Иначе RLS их заблокирует.
Итоговые RLS политики
Операция Политика Условие
SELECT Users can view own transactions auth.uid() = user_id
INSERT Users can create own transactions auth.uid() = user_id
UPDATE Users can update own transactions auth.uid() = user_id
DELETE Users can delete own transactions auth.uid() = user_id
RLS работает автоматически! Даже если кто-то попытается получить чужие данные через API — база данных вернёт только его транзакции.
