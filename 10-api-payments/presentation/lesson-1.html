Что такое Git и зачем он нужен

20 минут

Аналогия "Сохранение в игре"

Представьте, что вы играете в сложную игру:

  • Перед боссом вы сохраняетесь
  • Если проиграли - загружаете сохранение
  • У вас может быть несколько слотов сохранений

Git работает точно так же, но для кода!

Git термин Игровая аналогия
Commit Одно сохранение
Branch Слот сохранения / параллельная вселенная
Checkout Загрузить сохранение
Push Отправить сохранения в облако
Pull Скачать сохранения из облака
Merge Объединить два прохождения
Repository Папка со всеми сохранениями
GitHub Облачное хранилище сохранений

Зачем Git вайбкодеру?

Что такое Push и Pull

Push и Pull — это "мост" между вашим компьютером и GitHub:

Push

Отправить код на GitHub

Компьютер → Облако

Pull

Скачать код с GitHub

Облако → Компьютер

Commit vs Push — в чём разница?

Commit без Push = Сохранили файл, но НЕ загрузили в облако. Если компьютер сломается — код пропадёт!

Commit + Push = Сохранили файл И загрузили в облако. Код в безопасности!

Полный цикл работы:

1. Написал код
2. Commit  → Сохранил локально (на компьютере)
3. Push    → Отправил в облако (на GitHub)

На другом компьютере:
4. Pull    → Скачал из облака
5. Написал код
6. Commit + Push → Отправил обратно

Демонстрация: проблема без Git

Сценарий 1: AI сломал код

До:    Рабочее приложение
После: Белый экран, 50 ошибок

Без Git: "Всё пропало! Начинаем заново..."
С Git:   git checkout . -> Всё работает за 1 секунду

Сценарий 2: Хочу попробовать новую идею

Без Git: "А вдруг сломаю? Лучше не буду..."

С Git:
1. Создаю ветку
2. Экспериментирую
3. Получилось? Merge!
4. Не получилось? Удаляю ветку, main не пострадал

Вопрос 1: Что такое commit в Git?

Папка с файлами
Сохранение текущего состояния кода
Облачный сервер
Команда для удаления кода

Вопрос 2: Что такое Push?

Сохранение файла на компьютере
Скачивание кода из интернета
Отправка commits на GitHub (из компьютера в облако)
Удаление файлов

Вопрос 3: Чем Commit отличается от Push?

Ничем, это одно и то же
Commit сохраняет локально, Push отправляет на GitHub
Push сохраняет локально, Commit отправляет на GitHub
Оба отправляют код на GitHub
