Webhooks и статус подписки

30 минут

Что такое Webhooks

Проблема: Пользователь заплатил, но ваш сервер не знает об этом!

Без webhooks:
Вы постоянно спрашиваете Stripe: "Мне заплатили? А сейчас?"

С webhooks:
Stripe сама звонит вам: "Эй! Кто-то заплатил!"

Важные события:

СобытиеКогда
checkout.session.completedПлатёж прошёл
customer.subscription.deletedПодписка отменена
invoice.payment_failedПлатёж не прошёл

Создание Webhook endpoint

Промпт для Cursor
Создай API route /api/stripe/webhook:

1. Метод: POST
2. Получает raw body
3. Проверяет подпись Stripe
4. Использует STRIPE_WEBHOOK_SECRET
5. Обрабатывает события:
   - checkout.session.completed: логирует успех
   - customer.subscription.deleted: логирует отмену
6. Возвращает 200 OK

Настройка Stripe CLI

Установка:

# macOS
brew install stripe/stripe-cli/stripe

# Windows (scoop)
scoop install stripe

Настройка:

stripe login
# Откроется браузер — разрешите доступ

Запуск:

stripe listen --forward-to localhost:3000/api/stripe/webhook

# Копируем whsec_... и добавляем в .env.local:
STRIPE_WEBHOOK_SECRET=whsec_ваш_секрет

Тестирование

Терминал 1: npm run dev
Терминал 2: stripe listen --forward-to localhost:3000/api/stripe/webhook

Делаем тестовый платёж → Смотрим консоль → "Платёж прошёл!"
