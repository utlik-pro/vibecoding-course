Оптимизация и полировка

Доводим UI до production-ready состояния

Заметка для преподавателя

Финальная полировка — то, что отличает любительский проект от профессионального.

Цель: Lighthouse score > 85, работает на всех устройствах, темная тема.

Промпт 1: Loading States
Промпт для Cursor 
Добавь loading состояния:

1. Skeleton для карточек (пульсирующий placeholder)
2. Skeleton для таблицы
3. Spinner для кнопок при отправке
4. Progress bar для загрузки страницы
5. Skeleton для графиков

Используй Tailwind animate-pulse или framer-motion.
Промпт 2: Error States
Промпт для Cursor 
Создай компоненты для ошибок:

1. ErrorBoundary - ловит ошибки React
2. Empty State - "Нет задач" с иллюстрацией и CTA
3. 404 страница - дружелюбная с кнопкой домой
4. Network Error - "Проверьте соединение" с retry
5. Form Error - красивый alert с описанием

Все с возможностью retry и понятным текстом.
Промпт 3: Responsive адаптация
Промпт для Cursor 
Проверь и исправь responsive:

Mobile (< 768px):
- Sidebar скрыт, открывается как drawer
- Таблица превращается в карточки
- Stats в 2 колонки
- Графики на всю ширину

Tablet (768-1024px):
- Sidebar collapsed по умолчанию
- Таблица с горизонтальным скроллом
- Stats в 2 колонки

Проверь все формы, модалы, dropdown на мобильных.
Промпт 4: Performance
Промпт для Cursor 
Оптимизируй производительность:

1. Lazy load для тяжёлых компонентов (графики)
2. Image optimization через next/image
3. Мемоизация списков с useMemo
4. Debounce для поиска
5. Virtual scroll для длинных списков

Проверь Lighthouse score, цель > 90.
Lighthouse: DevTools → Lighthouse → Generate report
Промпт 5: Accessibility
Промпт для Cursor 
Улучши accessibility:

1. Все интерактивные элементы доступны с клавиатуры
2. Focus visible стили
3. ARIA labels для иконок-кнопок
4. Skip to content ссылка
5. Correct heading hierarchy
6. Alt text для изображений
7. Color contrast минимум 4.5:1

Проверь через axe DevTools.
Промпт 6: Dark Mode
Промпт для Cursor 
Добавь поддержку тёмной темы:

1. Toggle в header
2. Сохранение в localStorage
3. System preference detection
4. Все компоненты адаптированы
5. Плавный transition при переключении

Используй Tailwind dark: классы.
