## Урок 3.2: Сравнение с Cursor

**Продолжительность:** 1 час

### Цели урока:
- Понять когда использовать Claude Code, а когда Cursor
- Изучить преимущества и недостатки каждого инструмента
- Создать стратегию использования инструментов

### Содержание:

1. **Сравнение возможностей (20 мин)**
   
   **Cursor преимущества:**
   - Полноценная IDE среда
   - Интеграция с файловой системой
   - Git интеграция
   - Расширения и плагины
   - Автодополнение в реальном времени
   - Работа с большими проектами
   
   **Claude Code преимущества:**
   - Более мощная модель ИИ (Claude 3.5 Sonnet)
   - Лучше понимает архитектуру и дизайн
   - Интерактивные Artifacts
   - Доступность с любого устройства
   - Отличные объяснения и обучение
   - Работа с изображениями

2. **Когда использовать каждый инструмент (25 мин)**
   
   **Claude Code лучше для:**
   - Планирования архитектуры проекта
   - Создания прототипов и концептов
   - Обучения и изучения новых концепций
   - Рефакторинга и оптимизации
   - Создания документации
   - Решения сложных алгоритмических задач
   - Дизайна системы и баз данных
   
   **Cursor лучше для:**
   - Ежедневной разработки
   - Работы с существующими проектами
   - Отладки и багфиксов
   - Быстрого автодополнения
   - Интеграции с Git
   - Работы с терминалом
   - Управления зависимостями

3. **Гибридный подход (15 мин)**
   - Планирование в Claude Code
   - Реализация в Cursor
   - Рефакторинг снова в Claude Code
   - Копирование кода между инструментами
   - Синхронизация через Git

### Практическая работа:
Спланировать в Claude Code архитектуру todo-приложения, затем реализовать в Cursor.

## Урок 3.3: Практика с Claude Code

**Продолжительность:** 2 часа

### Цели урока:
- Получить практический опыт работы с Claude Code
- Создать реальные компоненты и функции
- Научиться эффективно формулировать запросы

### Содержание:

1. **Генерация API-эндпоинта (30 мин)**
   
   **Задача:** Создать RESTful API для управления пользователями
   
   **Шаги:**
   - Запрос: "Создай Node.js Express API для управления пользователями с CRUD операциями"
   - Анализ сгенерированного кода
   - Добавление валидации данных
   - Интеграция с базой данных
   - Обработка ошибок
   - Документация API

   **Пример запроса:**
   ```
   Создай Node.js Express API со следующими возможностями:
   - CRUD операции для пользователей (создать, получить, обновить, удалить)
   - Валидация входящих данных
   - Middleware для логирования
   - Обработка ошибок
   - MongoDB интеграция с Mongoose
   - JWT аутентификация
   
   Структура пользователя:
   - id, email, name, password, role, createdAt, updatedAt
   ```

2. **Создание схемы базы данных (30 мин)**
   
   **Задача:** Спроектировать схему для e-commerce приложения
   
   **Шаги:**
   - Описание бизнес-требований
   - Генерация схемы SQL
   - Создание связей между таблицами
   - Индексы для производительности
   - Миграции базы данных
   
   **Пример запроса:**
   ```
   Создай схему базы данных PostgreSQL для интернет-магазина:
   
   Сущности:
   - Пользователи (клиенты и администраторы)
   - Продукты с категориями и тегами
   - Заказы с позициями
   - Корзина покупок
   - Отзывы и рейтинги
   - Платежи и доставка
   
   Требования:
   - Нормализация до 3NF
   - Индексы для быстрого поиска
   - Ограничения целостности
   - Drizzle ORM схема
   ```

3. **Написание тестов (30 мин)**
   
   **Задача:** Создать unit и integration тесты
   
   **Шаги:**
   - Тестирование API endpoints
   - Мокирование зависимостей
   - Тестирование бизнес-логики
   - Integration тесты с базой данных
   - Настройка test environment
   
   **Пример запроса:**
   ```
   Создай comprehensive test suite для API пользователей:
   
   - Unit тесты для всех функций
   - Integration тесты для endpoints
   - Тестирование валидации и ошибок
   - Mocking для внешних сервисов
   - Test fixtures и данные
   - Jest + Supertest setup
   - Покрытие кода >90%
   ```

4. **Рефакторинг и оптимизация (30 мин)**
   
   **Задача:** Улучшить существующий код
   
   **Шаги:**
   - Анализ производительности
   - Применение design patterns
   - Улучшение читаемости кода
   - Оптимизация запросов к БД
   - Кэширование
   
   **Пример запроса:**
   ```
   Проанализируй и оптимизируй этот код:
   [прикладываем код]
   
   Фокус на:
   - Производительность
   - Читаемость и maintainability  
   - Security best practices
   - Design patterns применение
   - Error handling
   - Logging и monitoring
   ```

### Практические упражнения:

**Упражнение 1: API для блога (20 мин)**
Создать полноценный API для блог-платформы с постами, комментариями, категориями.

## Полезные техники работы с Claude Code

### Эффективные запросы:
1. **Контекст сначала**: "Я разрабатываю SaaS для управления проектами..."
2. **Конкретные требования**: "Нужен API endpoint который..."
3. **Примеры**: "Как в GitHub API..."
4. **Ограничения**: "Используя только стандартную библиотеку..."
5. **Формат выхода**: "Верни в формате TypeScript интерфейса..."

### Шаблоны запросов:

**Для архитектуры:**
```
Спроектируй архитектуру для [тип приложения]
Требования: [список функций]
Технический стек: [список технологий]
Ограничения: [производительность, бюджет, время]
Масштаб: [количество пользователей, данных]
```

**Для кода:**
```
Создай [тип компонента] который:
- [функция 1]
- [функция 2]
- [функция 3]

Требования:
- Язык: [TypeScript/Python/etc]
- Стиль: [functional/OOP]
- Тестирование: [да/нет]
- Производительность: [требования]
```

**Для рефакторинга:**
```
Проанализируй и улучши этот код:
[код]

Фокус на:
- Performance
- Readability
- Security
- Best practices
- [специфичные требования]
```
