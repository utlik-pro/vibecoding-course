## Урок 1.1: Что такое Vibecoding (20 минут)

### Цели урока
- Понять, что такое vibecoding и почему это важно в 2025 году
- Познакомиться со стеком инструментов курса

### Содержание

#### 1. Что такое Vibecoding? (5 мин)
**Определение:** Vibecoding — это подход к разработке, где вы описываете ИИ, что хотите получить, вместо написания кода вручную.

**Ключевые принципы:**
- Описание намерения вместо ручного кодирования
- Итеративное улучшение через диалог с ИИ
- Фокус на результате, а не на синтаксисе
- «Vibe» = интуитивное понимание того, что нужно

**Почему это важно в 2025?**
- AI-инструменты достигли уровня production-ready
- Скорость разработки увеличивается в 5-10 раз
- Порог входа в разработку снижается
- Компании массово внедряют AI-assisted development

#### 2. Классика vs Vibecoding (5 мин)

| Классическая разработка | Vibecoding |
|-------------------------|------------|
| Изучаешь синтаксис месяцами | Описываешь задачу словами |
| Пишешь каждую строку вручную | AI генерирует 80% кода |
| Гуглишь ошибки | AI объясняет и исправляет |
| Один язык = один проект | Мультистек без проблем |
| Часы на boilerplate | Минуты на готовый каркас |

### Содержание

#### 1. Подключение GitHub (10 мин)

**Зачем GitHub?**
- Хранение кода в облаке
- История изменений
- Совместная работа
- Интеграция с Vercel для деплоя

**В lovable.dev:**

5. Выберите «Create new repository»

**Проверка:**

- Убедитесь, что код загрузился

#### 2. Подключение Supabase (15 мин)

**Зачем Supabase?**
- Настоящая база данных (PostgreSQL)
- Аутентификация пользователей
- Синхронизация между устройствами
- Realtime обновления

**Шаг 1: Создание проекта Supabase**
1. Откройте https://supabase.com
2. Войдите через GitHub
3. Нажмите «New Project»
4. Выберите организацию
5. Введите название: `task-manager-db`
6. Создайте **сильный пароль** (сохраните его!)
7. Выберите регион (ближайший к вам)
8. Нажмите «Create new project»
9. Подождите ~2 минуты

**Шаг 2: Создание таблицы tasks**

В Supabase Dashboard → SQL Editor:

```sql
-- Создание таблицы для задач
CREATE TABLE tasks (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  title TEXT NOT NULL,
  description TEXT,
  priority TEXT DEFAULT 'medium',
  completed BOOLEAN DEFAULT FALSE,
  due_date TIMESTAMP WITH TIME ZONE,
  user_id UUID REFERENCES auth.users(id)
);

-- Включаем Row Level Security
ALTER TABLE tasks ENABLE ROW LEVEL SECURITY;

-- Политика: пользователи видят только свои задачи
CREATE POLICY "Users can view own tasks" ON tasks
  FOR SELECT USING (auth.uid() = user_id);

CREATE POLICY "Users can create own tasks" ON tasks
  FOR INSERT WITH CHECK (auth.uid() = user_id);

CREATE POLICY "Users can update own tasks" ON tasks
  FOR UPDATE USING (auth.uid() = user_id);

CREATE POLICY "Users can delete own tasks" ON tasks
  FOR DELETE USING (auth.uid() = user_id);
```

**Шаг 3: Получение ключей**

В Supabase → Settings → API скопируйте:
- `Project URL` → SUPABASE_URL
- `anon public` key → SUPABASE_ANON_KEY

**Шаг 4: Интеграция с lovable.dev**

1. В lovable.dev откройте Settings → Integrations
2. Нажмите «Connect Supabase»
3. Вставьте URL и ключ
4. Попросите AI обновить код:

```
Обнови приложение для использования Supabase вместо localStorage.
Используй подключённую интеграцию Supabase.
Таблица называется 'tasks' со следующими полями:
- id (UUID)
- title (TEXT)
- description (TEXT)
- priority (TEXT)
- completed (BOOLEAN)
- due_date (TIMESTAMP)
- user_id (UUID)
```

### Цели урока
- Закоммитить и запушить изменения на GitHub
- Задеплоить приложение на Vercel
- Получить публичную ссылку
- Понять домашнее задание

### Содержание

#### 1. Коммит и Push на GitHub (5 мин)

**В терминале Cursor:**
```bash
# Проверяем статус
git status

# Добавляем все изменения
git add .

# Создаём коммит
git commit -m "feat: добавлены категории и улучшен мобильный дизайн"

# Пушим на GitHub
git push origin main
```

**Проверка:**
Откройте репозиторий на GitHub и убедитесь, что изменения появились.

#### 2. Деплой на Vercel (7 мин)

**Шаг 1: Подключение репозитория**
1. Откройте https://vercel.com
2. Войдите через GitHub
3. Нажмите «Add New Project»
4. Выберите репозиторий `task-manager`
5. Нажмите «Import»

**Шаг 2: Настройка переменных окружения**
В разделе «Environment Variables» добавьте:
- `NEXT_PUBLIC_SUPABASE_URL` = ваш Supabase URL
- `NEXT_PUBLIC_SUPABASE_ANON_KEY` = ваш anon key

**Шаг 3: Деплой**
1. Нажмите «Deploy»
2. Дождитесь сборки (~2 минуты)
3. Получите ссылку вида: `task-manager-xxx.vercel.app`

**Готово!** Ваше приложение доступно в интернете.
