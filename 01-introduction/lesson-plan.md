# Модуль 1: Введение в Vibecoding

## Урок 1.1: Vibecoding в действии

**Продолжительность:** 3 часа (180 минут)  
**Фокус:** 90 % времени — практические упражнения с ИИ-инструментами  
**Формат:** живой воркшоп с групповыми спринтами и демо-осмотров

### Цели урока
- прожить цикл «идея → промпт → результат → ревизия» на реальном мини-проекте;
- собрать базовый стек из 3+ ИИ-инструментов и понять их зону ответственности;
- настроить рабочий проект, выгрузить его в GitHub и оформить документирование;
- сформировать личные принципы vibecoding-подхода.

### Обязательные предварительные требования
- установлены Cursor (или другой AI-IDE), Node.js LTS, Git;
- активны аккаунты в двух ИИ-сервисах (например, Cursor + Claude Code);
- настроен GitHub (SSH или PAT).

## Детализация блоков

### 1. Check-in & Prompt Arena (0:00 – 0:15)
- Блиц-вопросы: «Что ожидаю от vibecoding?», «Где ИИ уже помогает?».
- Упражнение «Prompt vs Result»: каждый пишет свой промпт, отправляет в Cursor Chat, сравниваем ответы и улучшаем формулировки.

### 2. Философия на примерах (0:15 – 0:35)
- Разбираем три кейса (прототип, баг-фикс, документация) и показываем, как «описание намерения» заменяет часть ручного кода.
- Мини-задача: преобразовать классический план разработки в «vibe-flow» (описание → генерация → review).

### 3. Vibecoding Stack Gauntlet (0:35 – 1:05)
- Чек-лист инструментов: IDE, генераторы UI, API-конструкторы, тестовые ассистенты.
- Практика:
  1. Через ИИ чаты составить персональный стек (минимум 3 инструмента, роли + ограничения).
  2. Создать Notion/Sheet с колонками «Инструмент / Цель / Лимиты / Стоимость».
  3. Выполнить регистрацию/логин, проверить доступ к API-ключам.

### 4. Спидран приложения (1:10 – 2:05)
- Бриф: «Todo + таймер фокус-сессий» или другой мини-продукт.
- Этапы:
  1. Сформулировать user story и критерии готовности через ИИ (5 мин).
  2. Генерация каркаса компонента в Cursor Composer (15 мин).
  3. Улучшение бизнес-логики (редактирование задач, фильтры) через чат ИИ (15 мин).
  4. Tailwind стили + анимация по подсказкам ИИ (10 мин).
  5. Добавление localStorage/IndexedDB для state persistence (10 мин).
- По ходу ведём лог промптов: что сработало, что нет.

### 5. Showcase & Bug Bash (2:05 – 2:20)
- Разбиваемся на пары, обмениваемся репозиториями.
- Каждый через ИИ ищет баги/улучшения в чужом проекте, фиксит и делает короткий Loom/скрин.

### 6. GitHub Workflow Sprint (2:20 – 2:50)
- Настройка репозитория (README, лицензия, .gitignore) через Cursor.
- Генерация превью (сценарий autopilot: `npm run dev` + screenshot, загрузка в README).
- Практика «ИИ-коммит»: формируем сообщение коммита с помощью чата, пушим ветку, создаём Pull Request шаблон.

### 7. Дебриф и персональный playbook (2:50 – 3:00)
- Каждому студенту ИИ помогает оформить три тезиса: «Где vibecoding применю», «Какой стек беру», «Что автоматизирую завтра».
- Собираем общий mural/whiteboard с инсайтами.

---

## Материалы и инструменты
- `Notion` или `Coda` для фиксации стеков и промптов;
- `Cursor`, `Claude Code`, `v0.dev` (рекомендуемый минимум — две AI IDE);
- шаблон README + чек-лист деплоя (доступ в Notion-хабе);
- ссылка на таблицу «10 инструментов vibecoding» (обновляется после занятия).

---

## Артефакты к концу урока
1. Репозиторий с рабочим приложением, README и TODO-бэклогом улучшений.
2. Notion-страница «Vibecoding Stack» с выбранными инструментами.
3. Документ «Prompt Log» — 5+ промптов и улучшенные версии.
4. Персональный playbook внедрения vibecoding (формат: 3 шага, дедлайны).

## Домашнее задание
1. За 20 минут собрать новое мини-приложение (калькулятор/конвертер/генератор) целиком через ИИ и задокументировать шаги.
2. Сделать три улучшения основного проекта (например, drag-n-drop задач, тёмная тема) и описать в README, какие промпты использовались.
3. Подготовить 2 вопроса к следующему уроку: один про стек инструментов, другой про организацию команды в режиме vibecoding.

## Метрики успеха урока
- ≥90 % участников загрузили проект в GitHub до конца занятия.
- У каждого заполнен стек-инструментов и playbook в Notion.
- Среднее время генерации MVP ≤55 минут.
- По итоговой рефлексии ≥80 % студентов чувствуют уверенность «описать задачу ИИ и получить результат».
