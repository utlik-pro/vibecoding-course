# Практические задания курса "Vibecoding с нуля"

## Структура заданий

Каждый модуль курса содержит несколько типов заданий:
- **Практические упражнения** - короткие задания на конкретные навыки
- **Мини-проекты** - небольшие приложения для отработки концепций  
- **Итоговые проекты** - комплексные задания для закрепления модуля

## Модуль 1: Введение

### Упражнение 1.1: Философское эссе
**Цель:** Осмыслить подход vibecoding
**Задание:** Написать эссе на 200 слов "Как я представляю программирование будущего?"
**Время:** 30 минут

### Упражнение 1.2: Тестирование ИИ-инструментов
**Цель:** Сравнить разные ИИ-помощники
**Задание:** 
- Выбрать 2 инструмента из списка (Cursor, Claude Code, v0.dev)
- Создать одну и ту же простую функцию в каждом
- Написать сравнительный отзыв
**Время:** 45 минут

### Упражнение 1.3: Первое приложение
**Цель:** Продемонстрировать силу ИИ-подхода
**Задание:** Создать калькулятор с базовыми операциями за 15 минут
**Инструменты:** Любой ИИ-помощник на выбор
**Время:** 15 минут + 15 минут на анализ

### Упражнение 1.4: Git workflow
**Цель:** Освоить основы версионирования
**Задание:**
- Создать репозиторий для курса
- Загрузить todo-приложение
- Сделать 3 коммита с улучшениями
**Время:** 30 минут

### Итоговый проект: Простое веб-приложение
**Варианты на выбор:**
- Конвертер валют с актуальными курсами
- Генератор паролей с настройками
- Простая игра (крестики-нолики, guess number)
- Калькулятор чаевых

**Требования:**
- HTML + CSS + JavaScript
- Минимум 3 функции
- Адаптивный дизайн
- Валидация пользовательского ввода
- Сохранение в localStorage (если применимо)

**Время:** 2 часа
**Критерии оценки:**
- Функциональность (40%)
- UI/UX качество (30%) 
- Код качество (20%)
- Использование ИИ (10%)

## Система оценивания

### Критерии оценки упражнений:
- **Функциональность** (40%) - Работает ли как задумано?
- **Код качество** (30%) - Читаемость, structure, best practices
- **UI/UX** (20%) - Пользовательский опыт
- **ИИ использование** (10%) - Эффективность работы с ИИ-инструментами

### Уровни оценки:
- **Отлично** (90-100%) - Превосходит ожидания, дополнительные функции
- **Хорошо** (80-89%) - Соответствует всем требованиям
- **Удовлетворительно** (70-79%) - Основные требования выполнены
- **Нуждается в доработке** (<70%) - Не все требования выполнены

### Portfolio проект

По завершении каждого модуля студенты добавляют лучшие работы в свое portfolio на GitHub с:
- Detailed README для каждого проекта
- Live demos (deployed на Vercel/Netlify)
- Code documentation и комментарии
- Reflections о использованных ИИ-инструментах

Финальное portfolio должно содержать 8-10 проектов демонстрирующих full-stack навыки и эффективное использование ИИ в разработке.
