Настройка CLAUDE.md

Цели урока:

  • Понять назначение CLAUDE.md
  • Создать конфигурацию для Claude Code
  • Настроить правила проекта для ИИ-ассистента
Заметка для преподавателя

Важно: Claude Code — это CLI инструмент от Anthropic. CLAUDE.md автоматически загружается при запуске claude в папке проекта. Покажите разницу в работе с файлом и без.

1. Что такое CLAUDE.md

CLAUDE.md — это файл инструкций для Claude Code CLI. Когда вы запускаете claude в терминале, он автоматически читает этот файл и использует его как контекст.

Зачем нужен CLAUDE.md:

  • Контекст проекта — Claude понимает структуру и стек
  • Правила кодинга — соблюдает ваши конвенции
  • Важные файлы — знает, где что находится
  • Команды — понимает, как запускать проект
Разница с .cursorrules: CLAUDE.md для Claude Code CLI, .cursorrules для Cursor IDE. Оба файла нужны для полноценного вайбкодинга.
2. Структура CLAUDE.md

Создайте файл CLAUDE.md в корне проекта:

markdown 
# Project: My SaaS App

## Overview
This is a SaaS application built with Next.js 14 and modern stack.
The project uses vibecoding approach with AI-assisted development.

## Tech Stack
- **Framework:** Next.js 14 with App Router
- **Language:** TypeScript (strict mode)
- **Styling:** Tailwind CSS + shadcn/ui
- **Database:** Supabase (PostgreSQL)
- **Auth:** Supabase Auth
- **AI:** OpenAI API
- **Hosting:** Vercel

## Project Structure
```
src/
├── app/              # Next.js App Router pages
│   ├── api/          # API routes
│   ├── (auth)/       # Auth pages (login, register)
│   └── (dashboard)/  # Protected pages
├── components/       # React components
│   ├── ui/           # shadcn/ui components
│   └── features/     # Feature-specific components
├── lib/              # Utilities and clients
│   ├── supabase.ts   # Supabase client
│   └── openai.ts     # OpenAI client
└── types/            # TypeScript types
```

## Coding Guidelines

### Components
- Use Server Components by default
- Add 'use client' only for interactivity
- Keep components small and focused
- Use TypeScript interfaces for props

### API Routes
- Handle errors with try/catch
- Return proper status codes
- Validate input data
- Use typed responses

### Database
- Use Supabase client from src/lib/supabase.ts
- Never expose service role key to client
- Use Row Level Security (RLS)

## Commands
- `npm run dev` - Start development server
- `npm run build` - Production build
- `npm run lint` - Run ESLint

## Important Notes
- Environment variables are in .env.local (not committed)
- API keys must never be hardcoded
- Use @/ alias for imports from src/
3. Best Practices

Что включать в CLAUDE.md:

Текущие задачи

Добавляйте секцию с текущими задачами:

markdown 
## Current Tasks
- [ ] Implement user profile page
- [ ] Add payment integration
- [x] Setup authentication
Известные проблемы

Документируйте известные баги:

markdown 
## Known Issues
- Toast notifications don't work in Safari
- Dark mode toggle needs page refresh
Специфичные правила

Добавляйте правила для конкретного проекта:

markdown 
## Project-Specific Rules
- All prices are stored in cents (integer)
- User IDs are UUIDs from Supabase
- API responses follow { data, error } pattern
Совет: Обновляйте CLAUDE.md по мере развития проекта. Актуальный файл = лучшие результаты.
4. Работа с Claude Code

Запуск Claude Code:

bash 
# В папке проекта
cd my-saas-app
claude

Claude Code автоматически:

  • Прочитает CLAUDE.md
  • Поймет контекст проекта
  • Будет следовать вашим правилам
  • Сможет редактировать файлы

Примеры команд:

text 
# В Claude Code CLI:

> Создай компонент UserProfile с аватаром и именем

> Добавь API route для получения списка пользователей

> Исправь ошибку в файле src/components/Header.tsx
