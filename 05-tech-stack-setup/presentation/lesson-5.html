Промпт-инженеринг для вайбкодинга

Цели урока:

  • Научиться писать эффективные промпты
  • Создать шаблоны для типичных задач
  • Понять best practices вайбкодинга
Заметка для преподавателя

Ключевой урок: Это самый важный урок для вайбкодинга. Качество промптов напрямую влияет на качество генерируемого кода. Покажите примеры хороших и плохих промптов.

1. Основы промптов

Хороший промпт содержит четыре элемента:

Элемент Описание Пример
Контекст Что за проект, какой стек "В Next.js проекте с shadcn/ui..."
Задача Что нужно сделать "...создай компонент карточки"
Требования Конкретные ограничения "...с аватаром, именем и кнопкой"
Формат Как должен выглядеть результат "...используя TypeScript интерфейс"
Плохой промпт: "Сделай компонент"
Хороший промпт: "Создай компонент UserCard с использованием shadcn/ui Card. Показывай аватар, имя и email. Добавь кнопку редактирования."
2. Шаблоны промптов

Шаблон для компонентов:

template 
Создай [тип компонента] для [назначение].

Стек: Next.js 14, TypeScript, Tailwind CSS, shadcn/ui

Функциональность:
- [требование 1]
- [требование 2]
- [требование 3]

Props:
- [prop 1]: [тип] - [описание]
- [prop 2]: [тип] - [описание]

Используй существующие компоненты из src/components/ui/

Шаблон для API routes:

template 
Создай API route для [действие].

Endpoint: /api/[путь]
Метод: [GET/POST/PUT/DELETE]

Входные данные:
- [поле]: [тип]

Ответ:
- Успех: { data: [структура] }
- Ошибка: { error: string }

Требования:
- Использовать [Supabase/OpenAI/etc]
- Валидировать входные данные
- Обработать ошибки с правильными статусами

Шаблон для исправления багов:

template 
Ошибка: [описание ошибки]
Файл: [путь к файлу]

Что происходит:
[текущее поведение]

Что должно быть:
[ожидаемое поведение]

Код с ошибкой:
[фрагмент кода или строка]

Сообщение об ошибке (если есть):
[текст ошибки или stack trace]
3. Примеры промптов
Создание формы
prompt 
Создай форму регистрации пользователя.

Стек: Next.js 14, TypeScript, shadcn/ui, react-hook-form, zod

Поля формы:
- Имя (обязательное, min 2 символа)
- Email (обязательное, валидный email)
- Пароль (обязательное, min 8 символов)
- Подтверждение пароля (должно совпадать)

Функции:
- Валидация на клиенте с zod
- Показ ошибок под полями
- Кнопка Submit с loading состоянием
- onSubmit callback для отправки данных

Используй компоненты из shadcn/ui:
- Form, FormField, FormItem, FormLabel
- Input, Button
API с базой данных
prompt 
Создай CRUD API для управления задачами (todos).

Endpoints:
- GET /api/todos - список задач пользователя
- POST /api/todos - создать задачу
- PUT /api/todos/[id] - обновить задачу
- DELETE /api/todos/[id] - удалить задачу

Структура задачи:
- id: uuid
- title: string
- completed: boolean
- user_id: uuid
- created_at: timestamp

Требования:
- Использовать Supabase client из src/lib/supabase.ts
- Проверять user_id для безопасности
- Возвращать { data } при успехе
- Возвращать { error } при ошибке с кодом 400/500
Интеграция с ИИ
prompt 
Создай API route для генерации текста с OpenAI.

Endpoint: POST /api/generate

Входные данные:
- prompt: string (текст от пользователя)
- max_tokens?: number (по умолчанию 500)

Функциональность:
- Использовать OpenAI client из src/lib/openai.ts
- Модель: gpt-4o-mini
- Добавить системный промпт: "You are a helpful assistant"
- Вернуть сгенерированный текст

Обработка ошибок:
- 400: если prompt пустой
- 500: если OpenAI API недоступен
- Логировать ошибки в console.error
4. Best Practices вайбкодинга
  1. Будьте конкретны — чем точнее промпт, тем лучше результат
  2. Указывайте контекст — стек, существующие файлы, паттерны
  3. Разбивайте большие задачи — лучше 5 маленьких промптов, чем 1 огромный
  4. Проверяйте код — ИИ может ошибаться, всегда ревьюйте
  5. Итерируйте — уточняйте промпт, если результат не идеален
Совет: Создайте файл prompts.md в проекте с вашими шаблонами. Это ускорит работу и обеспечит консистентность.
