Готовность к вайбкодингу

Цели урока:

  • Проверить все настройки ИИ-инструментов
  • Протестировать работу Cursor и Claude Code
  • Убедиться в полной готовности к разработке
Заметка для преподавателя

Финальная проверка: Этот урок — контрольная точка. Каждый студент должен пройти весь чек-лист и продемонстрировать работу ИИ-инструментов.

1. Полный чек-лист готовности

Проект:

  • Next.js запускается локально (npm run dev)
  • npm run build проходит без ошибок
  • Деплой на Vercel работает

ИИ-инструменты:

  • .cursorrules файл создан и актуален
  • CLAUDE.md настроен с документацией проекта
  • prompts.md с шаблонами готов

Интеграции:

  • Supabase клиент подключен
  • OpenAI API настроен
  • Health check endpoint работает
2. Тестирование Cursor

Проверьте работу Cursor с вашими настройками:

Шаги тестирования:

  1. Откройте проект в Cursor
  2. Убедитесь, что .cursorrules загружен (Ctrl/Cmd + Shift + P > "Cursor: View Rules")
  3. Откройте Composer (Ctrl/Cmd + K)
  4. Попробуйте сгенерировать простой компонент

Тестовый промпт:

prompt 
Создай компонент HelloWorld, который показывает
приветствие с текущей датой. Используй shadcn/ui Card.
Ожидаемый результат: Cursor должен создать TypeScript компонент с использованием shadcn/ui Card, соблюдая правила из .cursorrules.
3. Тестирование Claude Code

Проверьте работу Claude Code CLI:

Шаги тестирования:

bash 
# Перейдите в папку проекта
cd my-saas-app

# Запустите Claude Code
claude

# В Claude Code выполните:
> Покажи структуру проекта

> Создай утилиту formatDate в src/lib/utils.ts

Что проверить:

  • Claude понимает контекст проекта из CLAUDE.md
  • Claude использует правильный стек (TypeScript, Next.js)
  • Claude создает файлы в правильных папках
  • Код соответствует стилю проекта
