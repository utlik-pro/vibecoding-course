CLAUDE.md - мозг проекта

30 минут

Что такое CLAUDE.md?

CLAUDE.md — это инструкция для Claude о вашем проекте.

Когда вы запускаете claude в папке, он автоматически читает CLAUDE.md и понимает:

  • Какой технологический стек используется
  • Какой стиль кода предпочитаете
  • Какие есть особые правила
  • Как устроен проект
Аналогия

CLAUDE.md = Должностная инструкция для сотрудника

Без инструкции: "Сделай что-нибудь хорошее"
С инструкцией: "Мы используем React + TypeScript, пишем тесты, комментарии на русском"

Автогенерация через /init

Самый простой способ создать CLAUDE.md:

Terminal
$ cd ваш-проект
$ claude
> /init

# Claude проанализирует проект и создаст CLAUDE.md

Что Claude определит автоматически:

  • Язык программирования
  • Фреймворки (React, Next.js, Vue, и т.д.)
  • Систему сборки (npm, yarn, pnpm)
  • Структуру папок
  • Паттерны кода

Структура CLAUDE.md

Базовая структура

# Project: SaaS Analytics Dashboard

## Tech Stack
- Next.js 14 (App Router)
- TypeScript
- Tailwind CSS
- Supabase

## Code Style
- Функциональные компоненты
- camelCase для переменных
- PascalCase для компонентов
- Комментарии на русском

## Project Structure
```
src/
├── app/          # Страницы
├── components/   # UI компоненты
├── lib/          # Утилиты
└── types/        # TypeScript типы
```

## Commands
- `npm run dev` - dev сервер
- `npm run build` - сборка
- `npm run lint` - проверка кода

## Rules
- Не используй any в TypeScript
- Всегда добавляй loading состояния
- Используй server components

Иерархия CLAUDE.md файлов

Claude читает CLAUDE.md файлы на разных уровнях:

РасположениеПрименение
~/.claude/CLAUDE.mdГлобальные правила для всех проектов
./CLAUDE.mdПравила для текущего проекта
./subfolder/CLAUDE.mdПравила для конкретной папки

Синтаксис @imports

Подключайте другие файлы в CLAUDE.md:

# Project: My App

## Style Guide
@docs/style-guide.md

## API Documentation
@docs/api.md

## Component Examples
@src/components/Button/README.md
@imports

Используйте @path/to/file чтобы подключить содержимое другого файла. Claude прочитает его как часть инструкций.

Команды которые Claude не угадает

## Commands
- `npm run dev` - запуск (порт 3000)
- `npm run test:watch` - тесты в watch режиме
- `npm run db:migrate` - миграции базы

Правила стиля

## Code Style
- Функциональные компоненты
- camelCase для переменных
- PascalCase для компонентов
- Комментарии на русском

Чего НЕ делать

## Don't
- Не используй any в TypeScript
- Не создавай файлы без спроса
- Не удаляй существующие комментарии
Совет

CLAUDE.md обновляется по мере развития проекта. Добавляйте правила, когда замечаете что Claude делает что-то не так.

Практика: CLAUDE.md для нашего проекта

Создаём CLAUDE.md для SaaS Analytics Dashboard:

# Project: SaaS Analytics Dashboard

## Overview
SaaS приложение с landing page и dashboard.
Модуль 11-12 курса VibeCoding.

## Tech Stack
- Next.js 14 (App Router)
- TypeScript
- Tailwind CSS
- Recharts для графиков
- Vercel для деплоя

## Code Style
- Функциональные компоненты с TypeScript
- Tailwind для стилей (не CSS modules)
- Именование файлов: kebab-case
- Компоненты: PascalCase

## Project Structure
```
src/
├── app/
│   ├── page.tsx           # Landing
│   ├── dashboard/
│   │   └── page.tsx       # Dashboard
│   └── layout.tsx
├── components/
│   ├── landing/
│   └── dashboard/
└── lib/
    └── utils.ts
```

## Commands
- `npm run dev` - localhost:3000
- `npm run build` - сборка
- `npm run lint` - линтинг

## Rules
- 'use client' только для интерактивности
- Все тексты на русском
- Добавляй loading.tsx для страниц

Проверьте себя

Для документации проекта
Чтобы Claude понимал контекст и правила проекта
Для хранения паролей
Для настройки git
