Иерархия настроек

Четыре уровня настроек (от высшего к низшему):

Уровень Файл Применяется
Managed Системные папки IT администраторы
User ~/.claude/settings.json Все ваши проекты
Project .claude/settings.json Для команды (в git)
Local .claude/settings.local.json Только для вас
Приоритет

Local > Project > User > Managed

Локальные настройки перезаписывают все остальные.

Интерактивная настройка:

Claude Code
> /permissions

Permissions — новый формат

Claude Code использует систему allow / deny / ask для управления разрешениями.

Файл .claude/settings.json

{
  "permissions": {
    "allow": [
      "Read",
      "Write",
      "Edit",
      "Bash(npm run *)",
      "Bash(git add *)",
      "Bash(git commit *)"
    ],
    "deny": [
      "Bash(rm -rf *)",
      "Read(./.env)"
    ],
    "ask": [
      "Bash(git push *)"
    ]
  }
}

Три типа правил:

Тип Что делает
allow Разрешить без вопросов
deny Заблокировать полностью
ask Спрашивать каждый раз

Порядок проверки:

deny → ask → allow (первое совпадение выигрывает)

Примеры правил:

Правило Что разрешает
Read Читать любые файлы
Read(./.env) Читать конкретный файл
Bash(npm run *) npm скрипты
Bash(git *) Git команды
Важно

Всегда добавляйте Read(./.env) в deny чтобы защитить секреты!

Выбор модели

Claude Code поддерживает разные модели:

Модель Когда использовать Скорость
Sonnet Быстрые задачи, рутина Быстро
Opus Сложные задачи, архитектура Медленнее
Haiku Очень простые задачи Очень быстро

Выбор модели:

Terminal
# В настройках
> /config -> Model

# Через флаг при запуске
$ claude --model opus
Рекомендация для вайбкодеров

Начинайте с Sonnet — быстрее и дешевле.
Переключайтесь на Opus для сложных задач (архитектура, рефакторинг).

Практика: Настройка проекта

1. Создаём .claude/settings.json

{
  "permissions": {
    "allow": [
      "Read",
      "Write",
      "Edit",
      "Bash(npm run *)",
      "Bash(git add *)",
      "Bash(git commit *)"
    ],
    "deny": [
      "Read(./.env)",
      "Bash(rm -rf *)"
    ],
    "ask": [
      "Bash(git push *)"
    ]
  }
}

2. Создаём .claude/settings.local.json

Для личных настроек (не попадёт в git):

{
  "model": "opus"
}

3. Добавляем в .gitignore

# Claude Code local settings
.claude/settings.local.json

Проверьте себя

Вопрос 1: Какой файл настроек имеет наивысший приоритет?

~/.claude/settings.json (глобальный)
.claude/settings.json (проектный)
.claude/settings.local.json (локальный)
