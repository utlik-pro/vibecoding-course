Что такое API?

Учимся "разговаривать" с бэкендом на примере Money Tracker

API нашего Money Tracker

Вот какой API мы создадим для приложения финансов:

Действие пользователя Метод Endpoint
Открыл приложение — показать все транзакции GET /api/transactions
Нажал "Добавить расход" POST /api/transactions
Изменил сумму транзакции PATCH /api/transactions/5
Удалил транзакцию DELETE /api/transactions/5
API — это "меню" команд для бэкенда. Endpoint — конкретный адрес команды. Все термины в глоссарии →
Практика 1: Вызываем настоящий API

Задание: Откройте DevTools (F12) → Console и выполните этот код:

JavaScript — скопируйте в Console 
// GET запрос — получить данные
fetch('https://jsonplaceholder.typicode.com/posts/1')
  .then(response => response.json())
  .then(data => console.log(data));

Что вы увидите в консоли:

Ответ от API (JSON) 
{
  "userId": 1,
  "id": 1,
  "title": "sunt aut facere...",
  "body": "quia et suscipit..."
}
Вы только что вызвали API! fetch() отправил GET запрос, сервер вернул данные в формате JSON.
Практика 2: Разные запросы

Задание: Попробуйте эти запросы в Console (по одному):

1. Получить список постов (как список транзакций):

GET — получить много записей 
fetch('https://jsonplaceholder.typicode.com/posts?_limit=3')
  .then(r => r.json())
  .then(data => console.log('Список:', data));

2. Создать новую запись (как добавить транзакцию):

POST — создать новое 
fetch('https://jsonplaceholder.typicode.com/posts', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    title: 'Моя транзакция',
    body: 'Расход 500 рублей на кофе'
  })
})
.then(r => r.json())
.then(data => console.log('Создано:', data));

3. Удалить запись:

DELETE — удалить 
fetch('https://jsonplaceholder.typicode.com/posts/1', {
  method: 'DELETE'
})
.then(() => console.log('Удалено!'));
jsonplaceholder — это тестовый API. Он не сохраняет данные по-настоящему, но показывает как работают запросы.

Меняет существующую запись

Удаляет запись навсегда

JSON — формат данных

Данные передаются в формате JSON. Вот как будет выглядеть транзакция:

JSON — одна транзакция 
{
  "id": 42,
  "amount": 1200,
  "type": "expense",
  "category": "Кафе",
  "description": "Обед с коллегами",
  "date": "2024-01-15"
}

Правила JSON:

  • Ключи в двойных кавычках: "amount"
  • Строки в двойных кавычках: "expense"
  • Числа без кавычек: 1200
  • Массив в квадратных скобках: [ {}, {} ]
