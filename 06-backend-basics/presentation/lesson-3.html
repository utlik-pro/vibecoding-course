Что такое база данных?

Проектируем таблицу для хранения транзакций Money Tracker

Практика 1: Проектируем таблицу

Задание: Представьте, что вам нужно записать свои финансы в Excel. Какие колонки вы бы создали?

Вот пример записи: "15 января потратил 1200 рублей на обед в кафе"

Какая информация здесь есть?

  • Дата: 15 января
  • Сумма: 1200
  • Тип: расход (не доход)
  • Категория: еда
  • Описание: обед в кафе
Вы только что спроектировали структуру таблицы! База данных — это просто такой же Excel, только мощнее.
Наша таблица transactions

Вот как будет выглядеть таблица в Supabase (как в Excel):

id amount type category description date
1 50000 income Зарплата Январь 2024-01-15
2 1200 expense Еда Обед в кафе 2024-01-16
3 350 expense Транспорт Такси 2024-01-16
4 15000 income Фриланс Проект для клиента 2024-01-17
Каждая строка — это одна транзакция. Каждая колонка — это свойство транзакции.

Типы данных — что можно хранить

Каждая колонка имеет тип — какие значения туда можно записать:

text

Любой текст

"Еда", "Транспорт"

numeric

Числа (с копейками)

1200, 99.50

date

Дата

2024-01-15

int8

Целые числа

1, 2, 3, 42

Наши поля и их типы:

Поле Тип Почему?
id int8 Уникальный номер — всегда целое число
amount numeric Сумма может быть с копейками: 99.50
type text "income" или "expense"
category text Название категории
description text Комментарий (может быть пустым)
date date Дата транзакции
Практика 3: Зачем нужен id?

Задание: Представьте, у вас две одинаковых транзакции:

id amount category date
7 500 Кафе 2024-01-20
8 500 Кафе 2024-01-20

Вопрос: как удалить только вторую транзакцию?

Ответ 
DELETE /api/transactions/8

// Без id мы бы не смогли отличить одну запись от другой!
// id — это как номер паспорта для каждой записи.
id (Primary Key) — уникальный номер каждой записи. Создаётся автоматически: 1, 2, 3... Подробнее в глоссарии →
