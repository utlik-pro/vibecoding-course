Тестирование API

Цели урока:

  • Установить Thunder Client для тестирования
  • Протестировать все 4 операции Money Tracker
  • Создать коллекцию запросов для проекта
  • Научиться находить и исправлять ошибки
Зачем тестировать API?

Сценарий: Вы написали весь код для Money Tracker. Как убедиться, что API работает правильно до подключения фронтенда?

Тестирование через браузер работает только для GET. Для POST, PATCH, DELETE нужен специальный инструмент.

Thunder Client — расширение для VS Code/Cursor, позволяет отправлять любые HTTP запросы. Это как личный "тестировщик" API прямо в редакторе.
Практика 1: Установка Thunder Client

Задача: Установить инструмент для тестирования API Money Tracker.

Шаги:

  1. Откройте Cursor
  2. Нажмите Cmd+Shift+X (Extensions)
  3. В поиске введите: Thunder Client
  4. Нажмите Install на первом результате
  5. В левой панели появится иконка молнии ⚡

Проверка установки:

Кликните на иконку молнии — откроется панель Thunder Client.

Важно! Убедитесь что Next.js запущен: npm run dev
Практика 2: Тестируем получение транзакций (GET)

Сценарий: Пользователь открывает Money Tracker. Приложение должно загрузить все его транзакции.

Создаём первый запрос:

  1. Откройте Thunder Client (иконка молнии)
  2. Нажмите New Request
  3. Метод: GET (уже выбран по умолчанию)
  4. URL: http://localhost:3000/api/transactions
  5. Нажмите Send
Что вы должны увидеть? (клик) ▼

Status: 200 OK

Response:

[
  {
    "id": 1,
    "amount": 50000,
    "type": "income",
    "category": "Зарплата",
    "description": "За декабрь",
    "date": "2024-12-10"
  },
  {
    "id": 2,
    "amount": 1200,
    "type": "expense",
    "category": "Кафе",
    "description": "Обед",
    "date": "2024-12-11"
  }
  // ... остальные транзакции
]

Тестируем фильтрацию:

Теперь получим только расходы:

  1. URL: http://localhost:3000/api/transactions?type=expense
  2. Send
Какой будет ответ? ▼

Только транзакции с "type": "expense" — Кафе, Такси, Продукты.

Транзакции с type: "income" не появятся!

Практика 3: Добавляем новую транзакцию (POST)

Сценарий: Пользователь потратил 800 рублей на такси и хочет записать этот расход.

Создаём POST запрос:

  1. В Thunder Client нажмите New Request
  2. Смените метод с GET на POST
  3. URL: http://localhost:3000/api/transactions
  4. Перейдите во вкладку Body
  5. Выберите JSON
  6. Вставьте данные:
json 
{
  "amount": 800,
  "type": "expense",
  "category": "Транспорт",
  "description": "Такси до аэропорта"
}
  1. Нажмите Send
Ожидаемый результат (клик) ▼

Status: 201 Created

Response:

{
  "id": 8,
  "amount": 800,
  "type": "expense",
  "category": "Транспорт",
  "description": "Такси до аэропорта",
  "date": "2024-12-15",
  "created_at": "2024-12-15T10:30:00Z"
}

Заметьте: Транзакция получила id автоматически! Date тоже установилась на сегодня.

Проверка:

Выполните GET запрос ещё раз — новая транзакция должна быть в списке!

Практика 4: Исправляем ошибку (PATCH)

Сценарий: Оказалось, такси стоило 950 рублей, а не 800. Нужно исправить!

Важно!

Сначала узнайте id транзакции, которую хотите изменить. Допустим, это id = 8.

Создаём PATCH запрос:

  1. New Request
  2. Метод: PATCH
  3. URL: http://localhost:3000/api/transactions/8
  4. Body → JSON:
json 
{
  "amount": 950
}
  1. Send
Что должно вернуться? (клик) ▼

Status: 200 OK

Response:

{
  "id": 8,
  "amount": 950,
  "type": "expense",
  "category": "Транспорт",
  "description": "Такси до аэропорта",
  "date": "2024-12-15"
}

Важно: Изменилось только поле amount! Остальные поля сохранились.

Меняем несколько полей:

Можно изменить сразу несколько полей:

{
  "amount": 950,
  "description": "Такси до Шереметьево"
}
Практика 5: Удаляем транзакцию (DELETE)

Сценарий: Пользователь случайно добавил тестовую транзакцию и хочет её удалить.

Создаём DELETE запрос:

  1. New Request
  2. Метод: DELETE
  3. URL: http://localhost:3000/api/transactions/8
  4. Body: оставьте пустым (не нужен)
  5. Send
Результат удаления (клик) ▼

Status: 200 OK

Response:

{ "success": true }

Проверка:

  1. Выполните GET запрос на /api/transactions
  2. Транзакция с id 8 должна исчезнуть!
Осторожно! DELETE удаляет данные безвозвратно. В реальных приложениях часто делают "мягкое удаление" (добавляют поле deleted: true).
HTTP коды ответов в Money Tracker

Каждый ответ сервера содержит числовой код. Вот что они означают для нашего API:

Код Значение Пример в Money Tracker
200 OK GET вернул список транзакций
201 Created POST создал новую транзакцию
400 Bad Request POST с amount = -100 (нельзя!)
404 Not Found PATCH /api/transactions/999 (нет такой)
500 Server Error Ошибка в коде или Supabase недоступен
Как запомнить коды? (клик) ▼
  • 2xx — Всё хорошо (2 = "два больших пальца вверх")
  • 4xx — Ошибка клиента ("4" = "ты" опечатался)
  • 5xx — Ошибка сервера ("5" = сервер "свалился")
Практика 6: Создаём коллекцию запросов

Зачем? Чтобы не создавать запросы заново каждый раз. Сохраните их в коллекцию!

Создание коллекции:

  1. В Thunder Client перейдите в раздел Collections
  2. Нажмите New Collection
  3. Название: Money Tracker API

Сохранение запросов:

  1. Откройте любой созданный запрос
  2. Нажмите Save (Ctrl+S)
  3. Выберите коллекцию "Money Tracker API"
  4. Дайте понятное имя: "GET all transactions"

Создайте 5 запросов в коллекции:

Название Метод URL
GET all transactions GET /api/transactions
GET expenses only GET /api/transactions?type=expense
POST new transaction POST /api/transactions
PATCH update amount PATCH /api/transactions/1
DELETE transaction DELETE /api/transactions/1
Теперь вы можете тестировать API одним кликом! Коллекция сохраняется между сессиями.
Отладка: Что делать если ошибка?

Разберём типичные ошибки при тестировании Money Tracker API:

Ошибка: "Cannot read properties of null"

Причина: Supabase не подключен или ключи неверные.

Решение: Проверьте .env.local и перезапустите npm run dev.

Ошибка: 404 Not Found

Причина: Неправильный URL или файл route.ts отсутствует.

Решение: Проверьте путь: app/api/transactions/route.ts

Ошибка: Пустой массив [] при GET

Причина: Таблица transactions пуста или RLS блокирует.

Решение: Добавьте данные в Supabase и убедитесь что RLS выключен.

Ошибка: "Unexpected token" при POST

Причина: Неправильный JSON в Body.

Решение: Проверьте кавычки и запятые. JSON строгий!

Совет: Добавьте console.log() в API route и смотрите терминал — там будет подробная информация об ошибке.

Заметки преподавателя

Ключевые моменты:

  • Убедитесь что у всех запущен npm run dev ПЕРЕД началом урока
  • Покажите как найти id транзакции для PATCH/DELETE (сначала GET)
  • Демонстрируйте ошибки специально — пусть увидят 400, 404, 500
  • Акцентируйте внимание на коллекциях — это экономит время

Частые проблемы:

  • Забывают переключить метод (остаётся GET вместо POST)
  • Не выбирают JSON в Body
  • Используют одинарные кавычки в JSON (нужны двойные)
  • Путают / в конце URL (с / и без /)
