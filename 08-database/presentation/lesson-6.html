JOIN — объединяем таблицы

Получаем данные из нескольких таблиц одним запросом

Заметка для преподавателя

Важно: JOIN — один из самых важных концептов в SQL. Уделите время визуальной диаграмме!

Совет: Используйте аналогию со "склеиванием" таблиц. Две таблицы становятся одной по общему полю.

Что делает JOIN?

В прошлом уроке мы создали связь между tasks и categories. Но как получить название категории вместе с задачей?

Проблема 
-- Это даёт только ID категории, а не название!
SELECT * FROM tasks;

| id | title              | category_id |
|----|--------------------| ------------|
| 1  | Купить продукты    | 5           |  <-- Что такое 5?
| 2  | Позвонить маме     | 6           |  <-- Что такое 6?

JOIN "склеивает" две таблицы по общему полю:

Визуальная схема JOIN 
     TASKS                          CATEGORIES
+----+----------------+-----+     +----+----------+
| id | title          | c_id|     | id | name     |
+----+----------------+-----+     +----+----------+
| 1  | Купить продукты| 5   |---->| 5  | Покупки  |
| 2  | Позвонить маме | 6   |---->| 6  | Личное   |
| 3  | Сдать отчёт    | 1   |---->| 1  | Работа   |
+----+----------------+-----+     +----+----------+
              |                         |
              +-----------JOIN----------+
                          |
                          v
              РЕЗУЛЬТАТ ПОСЛЕ JOIN
+----+----------------+----------+
| id | title          | name     |
+----+----------------+----------+
| 1  | Купить продукты| Покупки  |  <-- Название вместо ID!
| 2  | Позвонить маме | Личное   |
| 3  | Сдать отчёт    | Работа   |
+----+----------------+----------+
Главное: JOIN соединяет строки из разных таблиц, когда значения в указанных колонках совпадают (tasks.category_id = categories.id).
Практика 1: Первый INNER JOIN

Задание: Получите задачи вместе с названиями категорий:

SQL 
SELECT tasks.title, categories.name
FROM tasks
INNER JOIN categories ON tasks.category_id = categories.id;

Разбор синтаксиса:

Часть запроса Что делает
SELECT tasks.title, categories.name Какие колонки берём (из обеих таблиц)
FROM tasks Основная таблица
INNER JOIN categories Присоединяем таблицу categories
ON tasks.category_id = categories.id Условие соединения (по какому полю)
Запустите запрос! Вы должны увидеть названия задач и категорий в одной таблице.
Практика 2: Псевдонимы таблиц (Aliases)

Писать tasks.title и categories.name долго. Используем короткие псевдонимы:

SQL 
-- t = tasks, c = categories (псевдонимы)
SELECT t.title, c.name
FROM tasks t
INNER JOIN categories c ON t.category_id = c.id;

Как работают псевдонимы:

  • FROM tasks t — теперь tasks можно писать как t
  • JOIN categories c — categories можно писать как c
  • Результат идентичен, но запрос короче!
Совет: Псевдонимы обычно — первая буква таблицы (t, c, u для users). Это стандартная практика.

Попробуйте: Добавьте больше полей в SELECT:

SQL 
SELECT t.id, t.title, t.priority, c.name, c.color
FROM tasks t
INNER JOIN categories c ON t.category_id = c.id;
Практика 3: LEFT JOIN — включаем все задачи

Проблема: INNER JOIN показывает только задачи, у которых ЕСТЬ категория. А если category_id = NULL?

SQL 
-- Добавим задачу без категории
INSERT INTO tasks (title, priority) VALUES ('Задача без категории', 'low');

-- INNER JOIN НЕ покажет эту задачу!
SELECT t.title, c.name
FROM tasks t
INNER JOIN categories c ON t.category_id = c.id;

Решение — LEFT JOIN:

SQL 
-- LEFT JOIN покажет ВСЕ задачи, даже без категории
SELECT t.title, c.name
FROM tasks t
LEFT JOIN categories c ON t.category_id = c.id;
Разница INNER vs LEFT JOIN 
INNER JOIN (только совпадения):      LEFT JOIN (все из левой таблицы):

| title              | name     |    | title                 | name     |
|--------------------|----------|    |-----------------------|----------|
| Купить продукты    | Покупки  |    | Купить продукты       | Покупки  |
| Позвонить маме     | Личное   |    | Позвонить маме        | Личное   |
| Сдать отчёт        | Работа   |    | Сдать отчёт           | Работа   |
                                      | Задача без категории  | NULL     | <-- Есть!
Когда что использовать:
  • INNER JOIN — когда связь обязательна (показать только задачи С категорией)
  • LEFT JOIN — когда нужны все записи из левой таблицы (все задачи)
Практика 4: Фильтрация с JOIN (WHERE)

Задание: Получите только задачи из категории "Работа":

SQL 
-- Фильтруем по названию категории
SELECT t.title, t.priority, c.name
FROM tasks t
INNER JOIN categories c ON t.category_id = c.id
WHERE c.name = 'Работа';

Попробуйте разные фильтры:

SQL 
-- Задачи с высоким приоритетом в любой категории
SELECT t.title, c.name, c.color
FROM tasks t
INNER JOIN categories c ON t.category_id = c.id
WHERE t.priority = 'high';

-- Несколько условий
SELECT t.title, c.name
FROM tasks t
INNER JOIN categories c ON t.category_id = c.id
WHERE c.name IN ('Работа', 'Учёба') AND t.completed = false;
Порядок: FROM -> JOIN -> WHERE. Сначала объединяем таблицы, потом фильтруем результат.
Практика 5: Сортировка с JOIN (ORDER BY)

Задание: Отсортируйте задачи по категории, затем по названию:

SQL 
-- Сортировка по названию категории
SELECT t.title, c.name AS category
FROM tasks t
INNER JOIN categories c ON t.category_id = c.id
ORDER BY c.name;

Комбинируем несколько сортировок:

SQL 
-- Сначала по категории, потом по приоритету
SELECT t.title, t.priority, c.name AS category
FROM tasks t
INNER JOIN categories c ON t.category_id = c.id
ORDER BY c.name ASC, t.priority DESC;

-- Можно сортировать и по полям из обеих таблиц!
SELECT t.title, c.name, c.color
FROM tasks t
INNER JOIN categories c ON t.category_id = c.id
ORDER BY c.color, t.title;
Практика 6: Реальный пример — полный список задач

Задание: Создайте запрос для отображения задач в приложении (как на реальном сайте):

SQL 
-- Полный запрос для UI приложения
SELECT
  t.id,
  t.title,
  t.description,
  t.completed,
  t.priority,
  t.created_at,
  c.name AS category_name,
  c.icon AS category_icon,
  c.color AS category_color
FROM tasks t
LEFT JOIN categories c ON t.category_id = c.id
WHERE t.completed = false
ORDER BY
  CASE t.priority
    WHEN 'high' THEN 1
    WHEN 'medium' THEN 2
    WHEN 'low' THEN 3
  END,
  t.created_at DESC;

Что делает этот запрос:

  • Получает все поля задачи + название, иконку и цвет категории
  • LEFT JOIN — показывает задачи даже без категории
  • WHERE — только невыполненные задачи
  • ORDER BY с CASE — сначала high, потом medium, потом low
  • Вторичная сортировка — новые задачи выше
Это реальный запрос! Именно так получают данные для списка задач в настоящих приложениях.
Мини-задание

Напишите запрос, который покажет:

  • Все задачи из категории "Учёба"
  • Поля: title, priority, category_name
  • Сортировка по дате создания (новые первыми)
Показать решение
SQL 
SELECT t.title, t.priority, c.name AS category_name
FROM tasks t
INNER JOIN categories c ON t.category_id = c.id
WHERE c.name = 'Учёба'
ORDER BY t.created_at DESC;
Проверка знаний

В чём разница между INNER JOIN и LEFT JOIN?

INNER JOIN быстрее, LEFT JOIN медленнее
INNER JOIN для двух таблиц, LEFT JOIN для трёх
INNER JOIN показывает только совпадения, LEFT JOIN — все записи из левой таблицы
Разницы нет, это синонимы
