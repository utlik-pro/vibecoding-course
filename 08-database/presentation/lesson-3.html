Сортировка и лимиты

ORDER BY, LIMIT, OFFSET — управляем порядком и количеством результатов

Практика 1: Сортировка ORDER BY

ORDER BY сортирует результаты по указанной колонке:

SQL 
-- Сортировка по названию (А-Я)
SELECT * FROM tasks ORDER BY title ASC;

-- Сортировка по названию (Я-А)
SELECT * FROM tasks ORDER BY title DESC;

-- Сначала новые задачи
SELECT * FROM tasks ORDER BY created_at DESC;

-- Сначала старые задачи
SELECT * FROM tasks ORDER BY created_at ASC;

ASC

По возрастанию

A-Z, 1-9, старые-новые

DESC

По убыванию

Z-A, 9-1, новые-старые

Совет: Если не указать ASC/DESC, по умолчанию используется ASC (по возрастанию)
Практика 2: Сортировка по нескольким колонкам

Можно сортировать сразу по нескольким полям — если значения в первом поле совпадают, SQL смотрит на второе:

SQL 
-- Сначала по приоритету, потом по дате создания
SELECT * FROM tasks
ORDER BY priority ASC, created_at DESC;

-- Сначала невыполненные, потом по приоритету
SELECT * FROM tasks
ORDER BY completed ASC, priority DESC;

-- Комбинация: статус, приоритет, название
SELECT * FROM tasks
ORDER BY completed ASC, priority DESC, title ASC;
Практический пример: В списке задач обычно сортируют так: сначала невыполненные (completed ASC), потом важные (priority DESC).
Практика 3: Ограничение результатов LIMIT

LIMIT ограничивает количество возвращаемых записей:

SQL 
-- Получить только 3 задачи
SELECT * FROM tasks LIMIT 3;

-- Топ-5 самых новых задач
SELECT * FROM tasks
ORDER BY created_at DESC
LIMIT 5;

-- Одна самая важная задача
SELECT * FROM tasks
WHERE priority = 'high' AND completed = false
ORDER BY created_at ASC
LIMIT 1;
Сценарий Запрос
Последние 10 заказов ORDER BY created_at DESC LIMIT 10
Самый дорогой товар ORDER BY price DESC LIMIT 1
5 случайных записей ORDER BY RANDOM() LIMIT 5
Практика 4: Пагинация с OFFSET

OFFSET пропускает первые N записей — это основа пагинации:

SQL 
-- Страница 1: первые 3 задачи
SELECT * FROM tasks
ORDER BY id
LIMIT 3 OFFSET 0;

-- Страница 2: следующие 3 задачи (пропустить первые 3)
SELECT * FROM tasks
ORDER BY id
LIMIT 3 OFFSET 3;

-- Страница 3: ещё 3 задачи (пропустить первые 6)
SELECT * FROM tasks
ORDER BY id
LIMIT 3 OFFSET 6;

Страница 1

LIMIT 3 OFFSET 0

Записи 1, 2, 3

Страница 2

LIMIT 3 OFFSET 3

Записи 4, 5, 6

Страница 3

LIMIT 3 OFFSET 6

Записи 7, 8, 9

Формула: OFFSET = (номер_страницы - 1) * LIMIT
Например, для страницы 5 по 10 записей: OFFSET = (5-1) * 10 = 40
Практика 5: Выборка конкретных колонок

Не всегда нужны все колонки. Выбирайте только необходимые:

SQL 
-- Только id и название для списка
SELECT id, title FROM tasks
ORDER BY created_at DESC;

-- Название и статус для отображения
SELECT title, completed FROM tasks
WHERE priority = 'high';

-- Комбинация: конкретные колонки + сортировка + лимит
SELECT id, title, priority FROM tasks
WHERE completed = false
ORDER BY priority DESC
LIMIT 5;
Почему это важно:
  • Меньше данных передаётся по сети
  • Быстрее работает запрос
  • Приложение получает только нужное
Полный пример: список задач с пагинацией

Типичный запрос для приложения со списком задач:

SQL 
-- Список задач: невыполненные сверху, важные выше, по 10 на страницу
SELECT id, title, priority, completed, created_at
FROM tasks
WHERE completed = false
ORDER BY
  CASE priority
    WHEN 'high' THEN 1
    WHEN 'medium' THEN 2
    WHEN 'low' THEN 3
  END,
  created_at DESC
LIMIT 10
OFFSET 0;
CASE в ORDER BY — продвинутый приём для кастомной сортировки. high=1, medium=2, low=3 — чем меньше число, тем выше в списке.
Проверка знаний

Как получить записи 11-20 для второй страницы (по 10 на странице)?

LIMIT 10 OFFSET 0
LIMIT 10 OFFSET 10
LIMIT 20 OFFSET 10
LIMIT 10 OFFSET 20
Мини-задания (делаем вместе)
  1. Получите 3 самые старые задачи
  2. Получите топ-5 задач отсортированных по названию (А-Я)
  3. Сделайте запрос для 3-й страницы (по 2 записи на странице)
Показать решения 
-- 1. Три самые старые задачи
SELECT * FROM tasks ORDER BY created_at ASC LIMIT 3;

-- 2. Топ-5 по названию А-Я
SELECT * FROM tasks ORDER BY title ASC LIMIT 5;

-- 3. Страница 3 (по 2 записи): OFFSET = (3-1) * 2 = 4
SELECT * FROM tasks ORDER BY id LIMIT 2 OFFSET 4;
