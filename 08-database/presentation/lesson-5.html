Связи между таблицами — Foreign Key

Создаём категории и связываем их с задачами

Заметка для преподавателя

Важно: Это ключевой урок! Связи между таблицами — фундамент реляционных баз данных.

Совет: Используйте аналогию с Excel: вместо повторения "Работа" в каждой строке, храним только номер категории.

Зачем нужны связи между таблицами?

Представим, что мы хотим добавить категории к задачам. Можно сделать так:

Плохой способ 
-- Категория как текст в каждой задаче
| id | title              | category  |
|----|--------------------| ---------|
| 1  | Купить продукты    | Дом       |
| 2  | Позвонить маме     | Дом       |
| 3  | Сдать отчёт        | Работа    |
| 4  | Оплатить интернет  | Дом       |
Проблемы такого подхода:
  • Дублирование данных (слово "Дом" повторяется 3 раза)
  • Опечатки ("Дом" vs "дом" vs "Домашние")
  • Сложно переименовать категорию (менять везде!)
  • Нельзя хранить доп. информацию о категории (цвет, иконка)

Решение: Создать отдельную таблицу для категорий!

Таблица categories

id, name, color, icon

Хранит информацию о категориях

Таблица tasks

... + category_id

Ссылается на категорию по id

Foreign Key — ключ к связям

Foreign Key (внешний ключ) — это колонка, которая ссылается на PRIMARY KEY другой таблицы.

Схема связи 
categories                    tasks
+----+--------+              +----+----------------+-------------+
| id | name   |              | id | title          | category_id |
+----+--------+              +----+----------------+-------------+
| 1  | Работа | <----------- | 1  | Сдать отчёт    | 1           |
| 2  | Дом    | <----------- | 2  | Купить продукты| 2           |
| 3  | Учёба  |              | 3  | Позвонить маме | 2           |
+----+--------+              +----+----------------+-------------+
     ^                                                   |
     |                                                   |
     +----------------- FOREIGN KEY ---------------------+
Что даёт Foreign Key:
  • Гарантирует, что category_id существует в таблице categories
  • Не даст удалить категорию, пока есть связанные задачи
  • Защита целостности данных на уровне базы
Практика 1: Создаём таблицу categories

Задание: Создайте таблицу для категорий задач:

SQL 
CREATE TABLE categories (
  id SERIAL PRIMARY KEY,
  name TEXT NOT NULL UNIQUE,
  color TEXT DEFAULT '#6B7280',
  icon TEXT DEFAULT 'folder'
);

Разбор полей:

Поле Тип Описание
id SERIAL PRIMARY KEY Уникальный номер категории (1, 2, 3...)
name TEXT NOT NULL UNIQUE Название (обязательное, уникальное)
color TEXT Цвет для UI (HEX-код)
icon TEXT Иконка для отображения
UNIQUE — гарантирует, что нельзя создать две категории с одинаковым именем.
Практика 2: Добавляем категории

Задание: Заполните таблицу тестовыми категориями:

SQL 
INSERT INTO categories (name, color, icon) VALUES
  ('Работа', '#EF4444', 'briefcase'),
  ('Дом', '#10B981', 'home'),
  ('Учёба', '#3B82F6', 'book'),
  ('Здоровье', '#F59E0B', 'heart'),
  ('Покупки', '#8B5CF6', 'shopping-cart'),
  ('Личное', '#EC4899', 'user');

Проверьте, что категории добавились:

SQL 
SELECT * FROM categories;
Должно появиться 6 категорий с id от 1 до 6.
Практика 3: Добавляем колонку category_id

Задание: Добавьте колонку для связи с категориями в таблицу tasks:

SQL 
-- Добавляем новую колонку в существующую таблицу
ALTER TABLE tasks
ADD COLUMN category_id INTEGER;

Синтаксис ALTER TABLE:

  • ALTER TABLE tasks — изменить таблицу tasks
  • ADD COLUMN — добавить колонку
  • category_id INTEGER — имя и тип колонки

Проверьте структуру таблицы:

SQL 
SELECT * FROM tasks;
Теперь в таблице tasks появилась колонка category_id со значением NULL для всех записей.
Практика 4: Создаём Foreign Key

Задание: Добавьте ограничение Foreign Key для защиты данных:

SQL 
-- Добавляем Foreign Key constraint
ALTER TABLE tasks
ADD CONSTRAINT fk_tasks_category
FOREIGN KEY (category_id)
REFERENCES categories(id);

Разбор команды:

Часть команды Что делает
ADD CONSTRAINT fk_tasks_category Создаём ограничение с именем fk_tasks_category
FOREIGN KEY (category_id) Указываем, какая колонка будет внешним ключом
REFERENCES categories(id) На какую таблицу и колонку ссылаемся
Теперь база данных защищена:
  • Нельзя записать несуществующий category_id
  • Нельзя удалить категорию, если есть связанные задачи
Практика 5: Связываем задачи с категориями

Задание: Присвоим категории существующим задачам:

SQL 
-- Сначала посмотрим id категорий
SELECT id, name FROM categories;

-- Работа = 1, Дом = 2, Учёба = 3, Здоровье = 4, Покупки = 5, Личное = 6

-- Присваиваем категории задачам
UPDATE tasks SET category_id = 5 WHERE title LIKE '%продукты%';
UPDATE tasks SET category_id = 6 WHERE title LIKE '%маме%';
UPDATE tasks SET category_id = 3 WHERE title LIKE '%домашк%';
UPDATE tasks SET category_id = 3 WHERE title LIKE '%курс%';
UPDATE tasks SET category_id = 4 WHERE title LIKE '%собак%';
UPDATE tasks SET category_id = 2 WHERE title LIKE '%интернет%';
UPDATE tasks SET category_id = 6 WHERE title LIKE '%книг%';

Проверьте результат:

SQL 
SELECT id, title, category_id FROM tasks;
Отлично! Теперь задачи связаны с категориями через category_id. В следующем уроке мы научимся объединять эти таблицы с помощью JOIN!
Проверяем работу Foreign Key

Попробуем нарушить целостность данных:

SQL 
-- Попытка добавить несуществующую категорию (id = 999)
INSERT INTO tasks (title, category_id)
VALUES ('Тестовая задача', 999);

-- Результат: ОШИБКА!
-- ERROR: insert or update on table "tasks" violates foreign key constraint
Foreign Key защитил данные! База не позволила создать задачу с несуществующей категорией.
Проверка знаний

Что гарантирует Foreign Key?

Автоматическое увеличение id
Уникальность значений в колонке
Что значение существует в связанной таблице
Что поле не может быть NULL
