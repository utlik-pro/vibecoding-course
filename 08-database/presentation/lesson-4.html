UPDATE и DELETE — изменение и удаление данных

Учимся безопасно обновлять и удалять записи в базе данных

ВНИМАНИЕ! UPDATE и DELETE — это "опасные" команды. Без WHERE они затронут ВСЕ записи в таблице. Всегда проверяйте запросы перед выполнением!
Практика 1: UPDATE одного поля

UPDATE изменяет существующие записи. Синтаксис:

SQL 
-- Отметить задачу с id=1 как выполненную
UPDATE tasks
SET completed = true
WHERE id = 1;

Разбор:

  • UPDATE tasks — какую таблицу обновляем
  • SET completed = true — какое поле меняем и на что
  • WHERE id = 1 — какую запись (ОБЯЗАТЕЛЬНО!)
SQL 
-- Проверяем результат
SELECT * FROM tasks WHERE id = 1;
Практика 2: UPDATE нескольких полей

Можно обновить сразу несколько полей через запятую:

SQL 
-- Обновляем название, описание и приоритет
UPDATE tasks
SET title = 'Срочно купить продукты',
    description = 'Молоко, хлеб, яйца, сыр',
    priority = 'high'
WHERE id = 1;
SQL 
-- Ещё пример: изменить приоритет и описание задачи 3
UPDATE tasks
SET priority = 'high',
    description = 'Срочно! Дедлайн завтра'
WHERE id = 3;
Совет: После каждого UPDATE делайте SELECT чтобы убедиться, что изменения верные.
Практика 3: Массовый UPDATE с WHERE

WHERE может выбрать несколько записей для обновления:

SQL 
-- Все задачи с низким приоритетом сделать средним
UPDATE tasks
SET priority = 'medium'
WHERE priority = 'low';

-- Проверяем: задач с low больше нет
SELECT * FROM tasks WHERE priority = 'low';
SQL 
-- Отметить все важные задачи как невыполненные (сброс)
UPDATE tasks
SET completed = false
WHERE priority = 'high';

-- Обновить все задачи с id от 1 до 3
UPDATE tasks
SET description = 'Обновлено!'
WHERE id IN (1, 2, 3);
После выполнения UPDATE вы увидите сообщение "Success. X rows affected" — сколько записей было изменено.
Практика 4: DELETE — удаление записи

DELETE удаляет записи из таблицы. Синтаксис:

SQL 
-- Удалить задачу с id = 7
DELETE FROM tasks
WHERE id = 7;

Разбор:

  • DELETE FROM tasks — из какой таблицы удаляем
  • WHERE id = 7 — какую запись (КРИТИЧЕСКИ ВАЖНО!)
SQL 
-- Удалить все выполненные задачи
DELETE FROM tasks
WHERE completed = true;

-- Удалить задачи с определёнными id
DELETE FROM tasks
WHERE id IN (5, 6);
Важно: DELETE безвозвратен! Удалённые данные нельзя восстановить (без бэкапа).
Практика 5: ОПАСНОСТЬ — что будет без WHERE
НЕ ВЫПОЛНЯЙТЕ ЭТИ КОМАНДЫ! Это только для демонстрации опасности.

Что произойдёт если забыть WHERE:

ОПАСНО - НЕ ВЫПОЛНЯТЬ! 
-- ОПАСНО: Обновит ВСЕ записи в таблице!
UPDATE tasks SET completed = true;
-- Все задачи станут выполненными

-- КАТАСТРОФА: Удалит ВСЕ записи!
DELETE FROM tasks;
-- Таблица станет пустой!
UPDATE без WHERE

Изменит ВСЕ записи

Все поля станут одинаковыми

DELETE без WHERE

Удалит ВСЕ записи

Таблица станет пустой

Правило: Перед UPDATE/DELETE ВСЕГДА проверяйте WHERE дважды!
Практика 6: Безопасный подход — SELECT перед UPDATE/DELETE

Золотое правило: Сначала SELECT, потом UPDATE/DELETE

SQL 
-- ШАГ 1: Сначала смотрим что будет затронуто
SELECT * FROM tasks WHERE priority = 'medium';

-- ШАГ 2: Убедились что это нужные записи? Только тогда UPDATE
UPDATE tasks SET priority = 'low' WHERE priority = 'medium';
SQL 
-- Пример для DELETE:

-- ШАГ 1: Смотрим что удалим
SELECT * FROM tasks WHERE completed = true AND id > 5;

-- ШАГ 2: Убедились? Удаляем
DELETE FROM tasks WHERE completed = true AND id > 5;
Лайфхак: Скопируйте WHERE часть из SELECT в UPDATE/DELETE. Так вы точно не ошибётесь!
Действие Безопасный способ
Обновить запись 1. SELECT ... WHERE ... 2. UPDATE ... WHERE ...
Удалить запись 1. SELECT ... WHERE ... 2. DELETE ... WHERE ...
Массовое изменение 1. SELECT COUNT(*) WHERE ... 2. Проверить число 3. Выполнить
Шпаргалка: UPDATE и DELETE
SQL 
-- UPDATE: изменение данных
UPDATE таблица SET поле = значение WHERE условие;
UPDATE таблица SET поле1 = знач1, поле2 = знач2 WHERE условие;

-- DELETE: удаление данных
DELETE FROM таблица WHERE условие;

-- RETURNING: вернуть изменённые/удалённые записи (PostgreSQL)
UPDATE tasks SET completed = true WHERE id = 1 RETURNING *;
DELETE FROM tasks WHERE id = 5 RETURNING *;
Проверка знаний

Что произойдёт при выполнении: DELETE FROM tasks;

Удалится одна случайная запись
Ничего не произойдёт, команда неполная
Удалятся ВСЕ записи из таблицы tasks
Удалится сама таблица tasks
Мини-задания (делаем вместе)

Сначала добавим тестовые данные:

SQL 
-- Добавляем задачи для практики
INSERT INTO tasks (title, description, priority, completed) VALUES
  ('Тестовая задача 1', 'Для практики UPDATE', 'low', false),
  ('Тестовая задача 2', 'Для практики DELETE', 'medium', false),
  ('Тестовая задача 3', 'Удалить потом', 'low', true);

Задания:

  1. Измените приоритет "Тестовая задача 1" на high
  2. Отметьте "Тестовая задача 2" как выполненную
  3. Удалите "Тестовая задача 3"
Показать решения 
-- 1. Изменить приоритет (сначала SELECT!)
SELECT * FROM tasks WHERE title = 'Тестовая задача 1';
UPDATE tasks SET priority = 'high' WHERE title = 'Тестовая задача 1';

-- 2. Отметить выполненной
SELECT * FROM tasks WHERE title = 'Тестовая задача 2';
UPDATE tasks SET completed = true WHERE title = 'Тестовая задача 2';

-- 3. Удалить задачу
SELECT * FROM tasks WHERE title = 'Тестовая задача 3';
DELETE FROM tasks WHERE title = 'Тестовая задача 3';
