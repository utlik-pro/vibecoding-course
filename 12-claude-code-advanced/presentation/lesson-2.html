Skills - свои команды для Claude

45 минут

Что такое Skills?

Аналогия

Skills - это как "рецепты" для Claude. Вместо того чтобы каждый раз объяснять что делать, вы записываете инструкцию один раз и вызываете её одной командой /имя.

Без Skills:

Вы: "Запусти тесты, потом собери проект, потом задеплой на Vercel,
     и верни мне ссылку. Используй npm run test, npm run build,
     vercel --prod"

С Skills (один раз написали инструкцию):

Вы: /deploy
Claude: Выполняю деплой... Готово! URL: myapp.vercel.app

Два типа контента Skills:

ТипЧто делаетПример
ReferenceЗнания, которые Claude применяет автоматическиСтиль кода, правила API, конвенции
TaskКоманда, которую вы вызываете через /имя/deploy, /test, /review

Встроенные Skills

Claude Code уже идёт с полезными Skills из коробки:

КомандаЧто делает
/simplifyПроверяет ваш код на качество, повторы и эффективность. Запускает 3 агента параллельно!
/batchМассовые изменения в проекте. Разбивает работу на части и делает параллельно
/debugДиагностика текущей сессии Claude Code
/loopПовторяет команду по расписанию (например, каждые 5 минут проверять деплой)
Примеры встроенных Skills

/simplify - Написали новый модуль? Запустите /simplify и три агента параллельно проверят: повторяющийся код, качество, эффективность. Результат - чистый код.

/batch - /batch migrate src/ from JavaScript to TypeScript - Claude сам разобьёт на задачи, создаст для каждой отдельную ветку, и сделает всё параллельно!

/loop - /loop 5m check if deploy succeeded - каждые 5 минут проверяет статус деплоя и уведомляет вас.

Как создать свой Skill

Структура файлов:

.claude/skills/
├── deploy/
│   └── SKILL.md       -> вызов через /deploy
├── test/
│   └── SKILL.md       -> вызов через /test
└── review/
    └── SKILL.md       -> вызов через /review

Формат SKILL.md:

Файл состоит из двух частей: Frontmatter (шапка между ---) и Содержимое (инструкции для Claude).

---
name: deploy
description: Задеплоить приложение на Vercel
disable-model-invocation: true
---

Задеплой приложение:

1. Запусти тесты: `npm run test`
2. Если тесты прошли, собери: `npm run build`
3. Задеплой: `vercel --prod`
4. Верни ссылку на деплой

Важные настройки (Frontmatter):

НастройкаЧто делаетПример
nameИмя командыdeploy -> /deploy
descriptionКогда использовать (Claude читает это!)"Deploy app to Vercel"
disable-model-invocationТолько ручной вызовtrue для /deploy
user-invocableСкрыть из меню /false для фоновых знаний
allowed-toolsРазрешённые инструментыRead, Grep, Bash
context: forkЗапуск в отдельном субагентеДля изолированных задач
Важное правило

disable-model-invocation: true - используйте для команд с "побочными эффектами": деплой, коммит, отправка сообщений. Иначе Claude может случайно их вызвать сам!

Примеры популярных Skills

/deploy - тесты → сборка → деплой на Vercel, одной командой
/review - проверка кода на баги, безопасность и стиль
/fix-issue 42 - берёт issue из GitHub и автоматически чинит
/pr-summary - генерирует описание для пул-реквеста
/migrate-component Button React Vue - мигрирует компонент между фреймворками
/changelog - генерирует список изменений из коммитов

Переменные и аргументы

$ARGUMENTS - всё что вы пишете после команды:

---
name: fix-issue
description: Починить баг из GitHub issue
disable-model-invocation: true
---

Почини баг из GitHub issue $ARGUMENTS:

1. Посмотри описание: `gh issue view $ARGUMENTS`
2. Найди и пойми проблему
3. Исправь баг
4. Напиши тесты
5. Создай коммит

Использование:

/fix-issue 42
# Claude получит: "Почини баг из GitHub issue 42..."

Динамический контекст:

!`команда` - выполняется ДО того как Claude увидит инструкцию:

---
name: pr-summary
description: Описание пул-реквеста
context: fork
agent: Explore
---

## Данные PR
- Изменения: !`gh pr diff`
- Файлы: !`gh pr diff --name-only`

## Задание
Сделай краткое описание этого пул-реквеста.
Отдельные аргументы

$ARGUMENTS[0] или $0 - первый аргумент
$ARGUMENTS[1] или $1 - второй аргумент
Пример: /migrate-component SearchBar React Vue
