Custom Skills - свои команды

45 минут

Что такое Skills?

Skills — это переиспользуемые команды для Claude Code.

Пример

/deploy → SKILL.md с инструкциями для деплоя
/test → SKILL.md с инструкциями для тестов
/review → SKILL.md с инструкциями для code review

Преимущества:

  • Сохраняют промпты в файлах
  • Можно шарить между проектами
  • Hot reload — обновляются без перезапуска

Структура SKILL.md

Расположение:

.claude/skills/
├── deploy/
│   └── SKILL.md
├── test/
│   └── SKILL.md
└── review/
    └── SKILL.md

Формат файла:

---
name: deploy
description: Deploy the application to production
disable-model-invocation: true
allowed-tools: Bash(npm *), Bash(vercel *)
---

Deploy the application:

1. Run the test suite
2. Build with `npm run build`
3. Deploy with `vercel --prod`
4. Return the deployment URL

Frontmatter опции:

ОпцияОписание
nameИмя = /команда
descriptionКогда использовать
disable-model-invocationТолько ручной вызов
allowed-toolsРазрешённые инструменты
context: forkЗапуск в subagent
agentТип агента (Explore, Plan)
Важно: disable-model-invocation

Используйте disable-model-invocation: true для skills с side effects: деплой, коммит, отправка сообщений.

Примеры полезных Skills

Skill: /deploy (с защитой)

---
name: deploy
description: Deploy to production
disable-model-invocation: true
allowed-tools: Bash(npm *), Bash(vercel *)
---

Deploy $ARGUMENTS to production:

1. Run `npm run build`
2. If successful, run `vercel --prod`
3. Return the deployment URL

Skill: /review (в subagent)

---
name: review
description: Code review with security focus
context: fork
agent: Explore
---

Review the code for:
1. Security issues (injection, XSS)
2. Performance problems
3. Code style violations

Provide specific line references.

Skill: /fix-issue (с аргументами)

---
name: fix-issue
description: Fix a GitHub issue
disable-model-invocation: true
---

Fix GitHub issue $ARGUMENTS:

1. Use `gh issue view` to get details
2. Understand the problem
3. Implement the fix
4. Write tests
5. Create a commit
Переменные

$ARGUMENTS — всё после /команды
$ARGUMENTS[0], $0 — первый аргумент

Практика: Создаём Skills

Terminal
$ mkdir -p .claude/skills/deploy
$ mkdir -p .claude/skills/test

Тестируем:

Claude Code
$ claude
> /deploy
# Claude выполнит инструкции из SKILL.md
