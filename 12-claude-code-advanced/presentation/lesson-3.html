Субагенты - помощники Claude

45 минут

Что такое субагенты?

Аналогия

Представьте, что вы - руководитель проекта. Вместо того чтобы делать всё самому, вы отправляете помощников: одного - исследовать код, другого - спланировать архитектуру, третьего - написать тесты. Каждый работает самостоятельно в своём окружении и возвращает результат.

Зачем субагенты?

  • Сохраняют контекст - исследование не засоряет основной разговор
  • Специализация - каждый агент настроен под задачу
  • Параллельность - несколько агентов работают одновременно
  • Экономия - можно использовать быстрые дешёвые модели для простых задач

Встроенные субагенты

АгентМодельЧто делаетИнструменты

PlanКак у васИсследование для планированияТолько чтение
general-purposeКак у васСложные многошаговые задачиВсе инструменты
BashКак у васВыполняет команды в терминалеBash
Как Claude выбирает агента

Claude сам решает какого агента запустить. Для быстрого поиска по коду - Explore (Haiku). Для планирования - Plan. Для сложной работы с изменениями - general-purpose.

Как Claude использует субагентов:

Claude Code
> Как работает авторизация в проекте?

# Claude автоматически:
# 1. Запускает Explore агента (быстрый Haiku)
# 2. Агент ищет файлы auth, login, session
# 3. Читает и анализирует код
# 4. Возвращает краткое резюме в основной разговор

Создание своих субагентов

Способ 1: Через команду /agents (рекомендуется)

> /agents
# Create new agent -> User-level или Project
# Generate with Claude -> описать что нужно
# Выбрать инструменты и модель

Способ 2: Вручную - файл в .claude/agents/

# .claude/agents/code-reviewer.md
---
name: code-reviewer
description: Expert code reviewer. Use proactively after code changes.
tools: Read, Grep, Glob, Bash
model: sonnet
---

Ты - senior code reviewer. Проверяй код на:

1. Качество и читаемость
2. Безопасность (SQL injection, XSS)
3. Производительность
4. Покрытие тестами

Для каждой проблемы указывай:
- Уровень: Critical / Warning / Suggestion
- Файл и строку
- Как исправить

Настройки субагента (frontmatter):

НастройкаЧто делает
nameУникальное имя агента
descriptionКогда Claude должен его использовать
toolsДоступные инструменты (Read, Edit, Bash...)

Память агентов

Что это?

Агент может запоминать информацию между сессиями! Например, code-reviewer запоминает паттерны вашего кода и с каждым разом проверяет лучше.

---
name: code-reviewer
description: Reviews code for quality
memory: user
---

Ты - code reviewer. Записывай в свою память:
- Паттерны кода этого проекта
- Частые ошибки
- Архитектурные решения

Перед проверкой - загляни в свою память.

Где хранится память:

ScopeПапкаКогда использовать
user~/.claude/agent-memory/Знания для всех проектов (рекомендуется)
project.claude/agent-memory/Только для этого проекта (в git)
local.claude/agent-memory-local/Только для вас в этом проекте (не в git)

Передний и задний план

Foreground (передний)Background (задний)
РазговорБлокирует - ждёте результатВы продолжаете работать
РазрешенияСпрашивает у васЗапрашиваются заранее
Как запуститьПо умолчаниюCtrl+B или попросить "run in background"
Изоляция через worktree

Параметр isolation: worktree даёт агенту собственную копию репозитория (через git worktree). Агент может менять файлы не мешая вашей работе. Идеально для параллельной разработки!

Параллельное выполнение

Можно запустить несколько агентов одновременно!

Claude Code
> Параллельно:
1. Исследуй структуру API
2. Найди все компоненты с формами
3. Проверь конфигурацию TypeScript

# 3 Explore агента работают одновременно!
# Каждый в своём контексте, не мешая друг другу
Когда это полезно

Анализ большого проекта, подготовка к рефакторингу, аудит кодбейза, параллельный code review (безопасность + производительность + тесты).

Реальные сценарии

Подготовка к рефакторингу: "Параллельно: 1) найди все deprecated API, 2) найди дублирующийся код, 3) проверь покрытие тестами" - три Explore агента за минуту дадут полную картину.

Онбординг в проект: "Как работает этот проект? Расскажи про архитектуру" - Explore агент обойдёт весь код и вернёт краткое описание, не засоряя ваш контекст.

Безопасная правка: С isolation: worktree агент получит свою копию репозитория. Если что-то сломает - ваш код не пострадает.

Практика

Задание 1: Создать своего агента

# Через интерфейс:
> /agents
# Create new agent -> описать агента 
# Или вручную: .claude/agents/debugger.md
---
name: debugger
description: Debugging specialist for errors and test failures
tools: Read, Edit, Bash, Grep, Glob
---

Ты - эксперт по отладке. Когда находишь баг:
1. Воспроизведи ошибку
2. Найди причину (root cause)
3. Исправь
4. Проверь что работает

Задание 2: Параллельные субагенты

> Параллельно исследуй:
1. Все API endpoints
2. Все формы и их валидацию
3. Конфигурацию TypeScript

Задание 3: Агент с памятью

# .claude/agents/style-guide.md
---
name: style-guide
description: Enforces project code style
memory: project
tools: Read, Grep, Glob
model: haiku
---

Ты следишь за стилем кода. Запоминай:
- Какие паттерны используются в проекте
- Именование переменных и функций
- Структуру компонентов
