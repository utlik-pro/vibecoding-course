Best Practices и Tips

15 минут

Оптимизация промптов

Будь конкретен

Хорошо: "Добавь кнопку 'Сохранить' синего цвета (#3B82F6) справа от заголовка формы"

Плохо: "Добавь кнопку где-нибудь"

Указывай контекст

Хорошо: "В файле components/Header.tsx добавь..."

Плохо: "Добавь в хедер..." (какой хедер?)

Разбивай задачи

Хорошо: 1. Сначала создай структуру → 2. Потом стили → 3. Потом логику

Плохо: "Создай полное приложение за один промпт"

Управление контекстом

Полезные команды:

КомандаЧто делает
/clearОчистка контекста между задачами
/compactСжатие контекста без потери смысла
/rewindОткат к предыдущему checkpoint
Esc+EscБыстрый откат изменений

Когда использовать /clear:

  • После завершения большой задачи
  • При переключении между частями проекта
  • Когда Claude начинает путаться
Checkpoints

Claude автоматически сохраняет checkpoints. Используйте /rewind чтобы откатить изменения к предыдущему состоянию.

Работа с большими проектами

1. Разделяйте на модули

  • Отдельный CLAUDE.md для каждого модуля
  • Работайте с одним модулем за раз

2. Используйте субагенты

  • Explore для поиска
  • Plan для планирования
  • Параллельные задачи для анализа

3. Коммитьте часто

  • После каждой законченной фичи
  • Перед большими изменениями
  • Легче откатиться при проблемах

Headless режим

Запуск Claude без интерактивной сессии:

$ claude -p "Run tests and fix any failures"

$ claude -p "Generate API docs" --output docs/api.md

Использование в CI/CD:

# GitHub Actions
- name: Run Claude Code
  run: claude -p "Check for security issues"
Автоматизация

Headless режим идеален для pre-commit hooks, CI/CD pipelines, и автоматизированных проверок.

Параллельная работа

Git Worktrees:

$ git worktree add ../project-auth feature/auth
$ cd ../project-auth && claude

Запустите несколько Claude Code сессий в разных worktrees для параллельной работы над фичами.

Writer/Reviewer паттерн:

  • Writer: Пишет код
  • Reviewer: Проверяет код в отдельной сессии
