Best Practices и Troubleshooting

30 минут

Главный принцип

Дай Claude способ ПРОВЕРИТЬ свою работу

Плохо: "Напиши функцию валидации email"

Хорошо: "Напиши функцию validateEmail. Тест-кейсы: user@example.com - true, invalid - false, @empty.com - false. Запусти тесты после написания"

Claude может сам проверить, что его код работает. Без тестов он "надеется" что всё правильно. С тестами - знает точно.

Эффективные промпты

Будь конкретен

Плохо: "Почини баг в логине"

Хорошо: "Пользователи сообщают что логин падает после истечения сессии. Проверь auth flow в src/auth/, особенно обновление токенов. Сначала напиши падающий тест, потом исправь."

Указывай паттерны

Хорошо: "Посмотри как реализованы существующие виджеты. HotDogWidget.php - хороший пример. Сделай новый CalendarWidget по такому же паттерну."

Разбивай задачи

Хорошо: 1. Сначала создай структуру -> 2. Потом стили -> 3. Потом логику
Плохо: "Создай полное приложение за один промпт"

Используй CLAUDE.md

Запишите в CLAUDE.md стиль кода, структуру проекта, часто используемые команды. Claude будет читать это в начале каждой сессии - не нужно повторять.

Управление контекстом

КомандаКогда использовать
/clearМежду несвязанными задачами - чистый контекст
/compactКогда контекст большой, но задача та же
/rewindОткатить к предыдущему checkpoint
Esc+EscБыстрый откат последних изменений
--continueПродолжить последнюю сессию
--resumeПродолжить конкретную сессию
/contextПосмотреть что загружено в контекст
Золотые правила

/clear после каждой законченной задачи - не копите контекст
Субагенты для исследований - не засоряйте основной разговор
Коммитьте перед большими изменениями - легче откатиться
/simplify после реализации - встроенный Skill почистит код

Типичные ошибки

"Каша из задач"

Проблема: Много несвязанных задач в одной сессии

Решение: /clear между задачами

Бесконечные правки

Проблема: Claude делает 10 попыток и ходит по кругу

Решение: /clear + начать с лучшего промпта. Укажите конкретные файлы и тест-кейсы.

Переполненный CLAUDE.md

Проблема: Файл стал огромным, Claude игнорирует правила

Решение: Перенести детали в Skills (Reference-тип), оставить в CLAUDE.md только главное

Claude читает всё подряд

Проблема: Исследует весь проект вместо нужной папки

Решение: Указать конкретные файлы: "Посмотри src/auth/login.ts"

Память Claude Code

Claude запоминает информацию двумя способами:

CLAUDE.md (вы пишете)Auto Memory (Claude пишет)
ЧтоПравила, конвенции, архитектураПаттерны, команды, инсайты
ГдеКорень проекта (в git)~/.claude/projects/.../memory/
Кто управляетВыClaude автоматически
ПросмотрОткрыть файл/memory
Совет

Периодически проверяйте /memory - удаляйте устаревшее, оставляйте полезное. Auto Memory ограничена 200 строками.

Troubleshooting

Общая диагностика

$ claude --version
$ claude doctor
# Покажет проблемы с установкой и конфигурацией

MCP сервер не подключается

# Проверить статус:
> /mcp

# Переподключить:
$ claude mcp remove имя-сервера
$ claude mcp add --transport http имя-сервера url

Skills не срабатывает

  • Проверьте description - содержит ли ключевые слова запроса
  • Спросите: "Какие Skills доступны?"
  • Вызовите напрямую: /имя-skill

Повторные запросы разрешений

> /permissions
# Добавьте правила для часто используемых инструментов

Ошибки аутентификации

> /logout
$ claude
# Войдёт заново

Когда что использовать

Простая шпаргалка

"Claude должен всегда знать это" → CLAUDE.md
"Хочу команду /deploy" → Skill
"Нужно исследовать код, не засоряя контекст" → Субагент
"Нужны несколько агентов, которые обсуждают между собой" → Agent Team
"Подключить GitHub/Slack/БД" → MCP сервер
"Гарантированно форматировать после редактирования" → Hook
"Запустить Claude из скрипта" → Headless (claude -p)
"Поделиться навыками с командой" → Плагин
