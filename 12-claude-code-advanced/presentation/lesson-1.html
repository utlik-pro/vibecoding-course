MCP серверы - суперсилы Claude

45 минут

Что такое MCP?

MCP (Model Context Protocol) - открытый стандарт для подключения внешних инструментов к Claude.

Аналогия

MCP - это как USB-порт для Claude. Обычный Claude умеет читать и писать файлы. А через MCP он получает "суперсилы" - может работать с GitHub, базами данных, Figma, Slack и десятками других сервисов.

Без MCPС MCP
Читает/пишет файлы+ работает с GitHub (issues, PR)
Выполняет bash+ читает Figma дизайн
Поиск по проекту+ SQL запросы к базе
Работает только локально+ Notion, Sentry, Jira, Slack

Как подключить MCP сервер

Способ 1: HTTP сервер (облачный) - рекомендуется

Сервер работает в облаке. Вы просто подключаетесь к нему по URL. Как зайти на сайт - ввели адрес и работаете.

Terminal
# Подключить GitHub
$ claude mcp add --transport http github https://api.githubcopilot.com/mcp/

# Подключить Notion
$ claude mcp add --transport http notion https://mcp.notion.com/mcp

# Подключить Sentry (мониторинг ошибок)
$ claude mcp add --transport http sentry https://mcp.sentry.dev/mcp

Способ 2: Stdio сервер (локальный)

Программа, которая запускается на вашем компьютере. Как установить приложение - оно работает локально. Для баз данных, файловых систем.

Terminal
# Подключить PostgreSQL базу данных
$ claude mcp add --transport stdio db -- npx -y @bytebase/dbhub --dsn "postgresql://user:pass@localhost/mydb"
Порядок аргументов

Все опции (--transport, --env, --scope) идут ПЕРЕД именем сервера. Двойной дефис -- отделяет имя от команды запуска.

Управление серверами:

КомандаЧто делает
claude mcp listПоказать все подключённые серверы
claude mcp get githubДетали конкретного сервера
claude mcp remove githubУдалить сервер
/mcpСтатус серверов внутри Claude Code

Области видимости (Scopes)

Аналогия

Как настройки Wi-Fi - можно подключить для себя, а можно для всей семьи через роутер.

ScopeГде хранитсяДля кого
local (по умолчанию)~/.claude.jsonТолько вы, только этот проект
project.mcp.json (в git)Вся команда - файл попадает в репозиторий
user~/.claude.jsonВы во всех ваших проектах
# Подключить для всей команды (сохраняется в .mcp.json)
$ claude mcp add --scope project --transport http github https://api.githubcopilot.com/mcp/

Аутентификация

Многие облачные MCP серверы требуют входа в аккаунт (OAuth).

Claude Code
# 1. Добавляем сервер
$ claude mcp add --transport http sentry https://mcp.sentry.dev/mcp

# 2. Внутри Claude Code - аутентификация
> /mcp
# Выбираем сервер → Authenticate
# Откроется браузер для входа в аккаунт
Как это работает

Токены аутентификации хранятся безопасно и обновляются автоматически. Через /mcp можно также отозвать доступ (Clear authentication).

Реальные примеры

GitHub MCP: "Посмотри открытые issues с лейблом bug и создай для каждого ветку с фиксом" - Claude сам найдёт issues, прочитает описание, создаст ветки и исправит код!

Sentry MCP: "Какие ошибки были в продакшне за последние 24 часа? Почини самую частую" - Claude проанализирует логи, найдёт root cause и исправит.

PostgreSQL MCP: "Какие пользователи зарегистрировались сегодня? Создай отчёт" - Claude напишет SQL запрос и покажет результат.

Figma MCP: "Посмотри дизайн главной страницы и создай компонент по макету" - Claude прочитает стили и размеры из Figma.

Практика

Задание 1: Подключаем GitHub MCP

Terminal
# 1. Добавляем
$ claude mcp add --transport http github https://api.githubcopilot.com/mcp/

# 2. Запускаем Claude Code
$ claude

# 3. Аутентификация
> /mcp
# Выберите "Authenticate" для GitHub

# 4. Тестируем
> Покажи мои репозитории
> Создай issue: "Добавить тёмную тему"

Задание 2: Подключаем базу данных

# Если у вас есть PostgreSQL:
$ claude mcp add --transport stdio db -- npx -y @bytebase/dbhub --dsn "postgresql://user:pass@localhost/mydb"

# Теперь в Claude Code:
> Какие таблицы есть в базе?
> Покажи 10 последних заказов
