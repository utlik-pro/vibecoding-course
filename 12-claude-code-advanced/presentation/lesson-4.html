Hooks - автоматизация workflow

30 минут

Что такое Hooks?

Hooks — это скрипты, которые запускаются автоматически при определённых событиях.

HookКогда срабатывает
PreToolUseПеред выполнением инструмента
PostToolUseПосле выполнения
PermissionRequestПри запросе разрешения
SessionEndПри завершении сессии
Аналогия

Hooks = Git hooks, но для Claude Code

pre-commit в Git = PreToolUse в Claude

Конфигурация Hooks

В .claude/settings.json:

{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Edit",
        "command": "npm run lint:fix"
      }
    ],
    "PostToolUse": [
      {
        "matcher": "Bash(npm run build)",
        "command": "echo 'Build done!'"
      }
    ],
    "SessionEnd": [
      {
        "command": "git status"
      }
    ]
  }
}

Параметры:

  • matcher — когда запускать (название инструмента)
  • command — bash команда для выполнения

Полезные Hooks

Автоформатирование

{
  "PreToolUse": [
    {
      "matcher": "Edit",
      "command": "npm run format"
    }
  ]
}

Уведомление при завершении

{
  "SessionEnd": [
    {
      "command": "osascript -e 'display notification \"Done\" with title \"Claude\"'"
    }
  ]
}

Автоматический линтинг

{
  "PostToolUse": [
    {
      "matcher": "Write",
      "command": "npm run lint"
    }
  ]
}

Логирование

{
  "PostToolUse": [
    {
      "matcher": "*",
      "command": "echo \"$(date): Tool used\" >> .claude/actions.log"
    }
  ]
}

Команда /hooks

Используйте /hooks для интерактивной настройки:

> /hooks

# Откроется интерактивное меню настройки hooks

Практика

Добавьте в .claude/settings.json:

{
  "permissions": {
    "allow": ["Read", "Write", "Edit", "Glob", "Grep"]
  },
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit",
        "command": "npm run lint"
      }
    ]
  }
}
Обратите внимание

Используйте permissions вместо устаревшего allowedTools.
