Hooks - автоматизация workflow

40 минут

Что такое Hooks?

Аналогия

Hooks - это как автоматические правила в вашем доме:
"Когда я ухожу - выключить свет" (SessionEnd)
"Перед готовкой - помыть руки" (PreToolUse)
"После уборки - проветрить" (PostToolUse)
"Когда звонит телефон - уведомить" (Notification)

Главное отличие от CLAUDE.md: инструкции в CLAUDE.md - это рекомендации (Claude может "забыть" или проигнорировать). Hooks - это гарантированные действия, которые выполняются КАЖДЫЙ раз. Это не просьба - это правило, которое невозможно обойти.

Простой пример

В CLAUDE.md написано "форматируй код через Prettier". Claude может забыть.
Hook на PostToolUse с Prettier запустится автоматически после каждого редактирования - 100% гарантия.

Типы событий:

СобытиеКогда срабатываетПример использования
PreToolUseПЕРЕД использованием инструментаЗаблокировать изменение .env
PostToolUseПОСЛЕ использования инструментаАвтоформатирование кода
NotificationКогда Claude ждёт вашего ответаУведомление на экран
SessionStartПри запуске сессииЗагрузить контекст проекта
SessionEndПри завершении сессииОчистить временные файлы
StopКогда Claude закончил работуПроверить задачу
SubagentStartКогда субагент запускаетсяПодготовить окружение
SubagentStopКогда субагент завершаетсяОчистить ресурсы

Настройка Hooks

Способ 1: Через команду /hooks (рекомендуется)

> /hooks
# Интерактивный интерфейс:
# Выберите событие -> добавьте команду

Способ 2: В .claude/settings.json

Уведомление когда Claude ждёт (macOS):

{
  "hooks": {
    "Notification": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "osascript -e 'display notification \"Claude ждёт вас\" with title \"Claude Code\"'"
          }
        ]
      }
    ]
  }
}

Автоформатирование после редактирования:

{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "jq -r '.tool_input.file_path' | xargs npx prettier --write"
          }
        ]
      }
    ]
  }
}

Блокировка защищённых файлов:

#!/bin/bash
# .claude/hooks/protect-files.sh
INPUT=$(cat)
FILE=$(echo "$INPUT" | jq -r '.tool_input.file_path // empty')

# Список защищённых файлов
if [[ "$FILE" == *".env"* ]] || [[ "$FILE" == *"package-lock"* ]]; then
  echo "Заблокировано: $FILE защищён" >&2
  exit 2   # exit 2 = заблокировать действие
fi

exit 0     # exit 0 = разрешить
Коды выхода

exit 0 - разрешить действие
exit 2 - заблокировать действие (Claude увидит сообщение из stderr)

Три типа хуков

1. Command - запускает команду

Самый простой тип. Запускает shell-команду.

{
  "type": "command",
  "command": "osascript -e 'display notification \"Готово!\"'"
}

2. Prompt - спрашивает у AI

Отправляет промпт AI-модели для проверки:

{
  "type": "prompt",
  "prompt": "Проверь, выполнены ли все задачи. Если нет - ответь что осталось."
}

3. Agent - запускает агента

Самый мощный тип. Запускает полноценного агента для проверки:

{
  "type": "agent",
  "prompt": "Запусти тесты и проверь результаты",
  "timeout": 120
}
Когда какой использовать

Command - для простых действий (уведомления, форматирование)
Prompt - для проверки условий ("правильно ли?")
Agent - для сложных автоматических проверок

Реальные примеры хуков

Уведомление на Mac: Notification хук показывает системное уведомление когда Claude ждёт вашего ответа. Можно переключиться на другую работу и не пропустить.

Защита .env: PreToolUse хук проверяет, не пытается ли Claude редактировать .env файл, и блокирует действие (exit 2).

Авто-Prettier: PostToolUse хук автоматически форматирует файл после каждого редактирования. Claude пишет код, хук форматирует - красиво всегда!

Контроль качества: Stop хук с типом "agent" запускает проверку: "Все ли задачи выполнены? Есть ли ошибки?" Если нет - Claude продолжает работу.

Matcher - фильтрация событий

Matcher определяет ДЛЯ КАКИХ инструментов срабатывает хук:

MatcherЧто фильтрует
"" (пустой)Срабатывает на ВСЁ
"Edit|Write"Только при редактировании или записи файлов
"Bash"Только при выполнении bash-команд
"db-agent"Только для конкретного субагента (SubagentStart/Stop)

Практика

Задание 1: Настроить уведомления

> /hooks
# Выберите Notification -> добавьте команду уведомления

Задание 2: Защитить файлы

Создайте PreToolUse хук который блокирует изменение .env файлов:

> /hooks
# PreToolUse -> Edit|Write -> скрипт проверки

Задание 3: Автоформатирование

Настройте PostToolUse хук для автоматического запуска Prettier после каждого редактирования.
